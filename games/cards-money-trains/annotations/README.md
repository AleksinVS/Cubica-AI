# Аннотации авторской карты

Этот каталог хранит структурированные результаты переноса объектов с
авторских изображений. Аннотация — это проверяемый список координат и связей,
а не автоматически принятое игровое правило.

## Текущий результат

`initial-network.review.json` содержит черновое извлечение 25 точек и 10
дорог из полной карты `5079 × 3627`. Координаты сняты непосредственно с файла
`Начальная транспортная сеть.png`, который совпадает по размеру с базовой
картой. Открытые вопросы перечислены внутри файла в `reviewIssues`.

`initial-network.review-overlay.svg` — контрольное наложение тех же данных
поверх исходного изображения. Жёлтые пунктирные объекты означают
«распознано, но игровой статус ещё не подтверждён».

## Векторный черновик областей

`vector-map.review.json` — воспроизводимый, но непубликационный результат
чтения PDF-совместимой части файла `Карта Гвиней  а4.ai`. Извлекатель не
разбирает большой служебный блок Illustrator: он использует только открытый
поток рисования страницы и два его слоя — карту и терминалы.

Калибровка (преобразование координат одного изображения в координаты другого)
строится по 23 шестерёнкам терминалов. Их векторные центры автоматически
сопоставляются с уже проверяемыми точками `terminal-1`…`terminal-23`, после
чего вычисляется аффинное преобразование — масштаб, отражение, небольшой сдвиг
и допустимая коррекция перекоса. Параметры и ошибка сохранены внутри JSON.

В текущем верхнеуровневом потоке страницы найден ровно `981` оператор
обводки `S`; они сохраняются как `boundaryCandidates`. Вложенные изображения
и формы `Im0`/`Fm0`…`Fm4` не раскрываются и не считаются границами. Любое
изменение этого состава останавливает извлечение и требует нового разбора
источника. Это только кандидаты границ: у них намеренно нет
`regionId` и `countryId`. `vector-map.review-overlay.svg` показывает их
пурпурным цветом поверх карты, а бирюзовые метки позволяют визуально проверить
калибровку терминалов. Формат результата закреплён локальной JSON Schema
`vector-map-review.schema.json`; он не является контрактом runtime-манифеста.

## Статусы

- `review-draft` — можно проверять координаты и связи, но нельзя собирать
  runtime-манифест;
- `author-confirmed` — автор подтвердил смысл, начальные состояния и
  контрольное наложение;
- `mock` — заведомо вымышленные технические данные, не относящиеся к этой
  авторской игре.

Общий конвертер намеренно отказывается публиковать `review-draft` и
`template`, даже если файл технически корректен. Адаптер нормативной игры
дополнительно принимает только `author-confirmed`; статус `mock` может
собирать только отдельный mock-пакет. У публикуемого статуса не может остаться
ни одного `reviewIssue`.

## Как проверить

```bash
node games/cards-money-trains/tools/convert-map-annotation.mjs \
  --input games/cards-money-trains/annotations/initial-network.review.json \
  --overlay games/cards-money-trains/annotations/initial-network.review-overlay.svg

node games/cards-money-trains/tools/extract-vector-map-review.mjs --check
```

Команда проверяет общую JSON Schema (машиночитаемый контракт полей),
контрольную сумму исходного изображения, координаты, уникальность объектов и
ссылки дорог на существующие концы. Затем она заново строит контрольное
наложение.

Вторая команда повторно читает открытый PDF-поток `.ai`, проверяет единые
снимки и контрольные суммы AI-файла, опорной аннотации и PNG-фона, размеры
фона `5079 × 3627`, структуру слоёв, 23 соответствия терминалов, ошибку
калибровки, локальную JSON Schema и побайтовую актуальность двух векторных
review-артефактов. Для сознательной перегенерации нужно запустить ту же
команду без `--check`; оба производных файла заменяются через отдельные
синхронизированные временные файлы, а не записываются поверх существующих.

## Как опубликовать

До публикации нужно закрыть все `reviewIssues`, добавить подтверждённые
контуры областей, заменить неизвестные начальные состояния и получить
письменное подтверждение автора. Только после этого статус меняется на
`author-confirmed`, а производный фрагмент можно включить в authoring-вход
игры. Ручное копирование объектов в runtime-манифест запрещено.

При публикации общий конвертер нормализует каждый простой контур, безопасно
отклоняет перекрытия, выводит переходы только из точных общих границ
положительной длины и вычисляет контрольную сумму геометрии. Этот производный
граф используется принятым серверным планировщиком дороги; визуальное касание
или небольшой разрыв линий не исправляются автоматически.

Векторный черновик требует дополнительного шага: человек должен собрать из
кандидатов замкнутые области, назначить каждой области устойчивый идентификатор
и страну, проверить отверстия/разрывы и только затем перенести подтверждённые
полигоны в общую аннотацию карты. Сам `vector-map.review.json` не меняет статус
на `author-confirmed` и никогда напрямую не компилируется в runtime.
