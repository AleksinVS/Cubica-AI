## Аннотация транспортной сети

`convert-map-annotation.mjs` — тонкий игровой адаптер общего конвейера
аннотаций карты. Он подставляет правила именно этой игры, но не дублирует
общую проверку координат, ссылок и статусов.

## Векторный AI-источник

`extract-vector-map-review.mjs` читает PDF-совместимую часть файла
`draft/trains/Карта Гвиней  а4.ai`. Алгоритм ограничен текущим авторским
источником и выполняет следующие действия:

1. Читает страницу и её открытый сжатый поток рисования; служебные данные
   Illustrator не разбираются.
2. Находит в слое терминалов 23 повторяющиеся шестерёнки по их векторному
   примитиву, а не по закодированному тексту.
3. Сопоставляет центры шестерёнок с `terminal-1`…`terminal-23` из проверяемой
   аннотации начальной сети и вычисляет аффинное преобразование координат.
4. Извлекает ровно 981 верхнеуровневую обводку `S` в анонимные кандидаты;
   известные вложенные XObject не раскрываются. Новый XObject или другой
   оператор рисования останавливает обработку.
5. Проверяет PNG-фон по контрольной сумме и заголовку IHDR `5079 × 3627`.
6. Проверяет результат по локальной JSON Schema и создаёт контрольное SVG-
   наложение.

AI-файл, опорная JSON-аннотация и PNG-фон читаются как стабильные снимки:
инструмент сверяет файловый идентификатор, размер и время изменения до и после
чтения. Опорный JSON разбирается из тех же байтов, по которым вычислена его
контрольная сумма. Сжатый PDF-поток имеет жёсткий лимит распаковки 16 МиБ.

Перед записью проверяются обычные пути, символические ссылки и жёсткие ссылки
(два имени одного файла). JSON и SVG сначала полностью записываются в новые
временные файлы с исключительным созданием, синхронизируются с диском и только
затем атомарно переименовываются поверх производных файлов.

Сознательная перегенерация:

```bash
npm run generate:cards-money-trains-vector-review
```

Проверка, что производные файлы не устарели:

```bash
node games/cards-money-trains/tools/extract-vector-map-review.mjs --check
```

## Грузы и новости из XLSX

`import-cargo-news.mjs` создаёт воспроизводимый непубликационный снимок двух
листов `draft/trains/Колоди + Новости.xlsx`. Исходный XLSX не изменяется.
Вспомогательный `read-xlsx-rows.py` читает ограниченный и не содержащий формул
OpenXML-пакет стандартной библиотекой Python; сторонние пакеты не нужны.

Производный файл
`authoring/fixtures/cargo-news.intake.json` содержит:

- 174 исходные строки грузов с номером строки, терминалами, выплатой и
  принадлежностью базовой колоде либо добавлению новости;
- 34 новости и прямые связи новостей № 1–10 с добавляемыми строками грузов;
- контрольную сумму исходного XLSX;
- подтверждение, что каждая строка означает одну физическую карту, а
  одинаковые строки являются разными экземплярами;
- явные незакрытые вопросы по оставшимся эффектам новостей.

`build-card-lifecycle.mjs` переносит этот снимок в непубликационную игровую
сборку: создаёт 23 защищённые колоды терминалов, одноразовую колоду из 34
новостей, жизненный цикл выбора грузов и подтверждённые эффекты новостей
№ 1–10, 14, 16, 22–27 и 30–34. Новость № 14 включает прогрессивный налог до
конца партии или до его отмены другой новостью. Налог пересчитывается при
каждом завершении обслуживания: паровозная гильдия платит за активные
локомотивы сверх трёх, компания-перевозчик — за активные вагоны сверх пяти.
Недостаток денег отклоняет всё завершение обслуживания без частичных
списаний. Новость № 16 атомарно собирает по пять монет
только с команд, имевших больше пятнадцати монет в начале новостного этапа.
Новость № 22 включает на текущий ход сбор в одну монету с первого успешного
движения каждого локомотива; повторный переход того же локомотива не
оплачивается повторно. Временные эффекты сбрасываются при входе в новостной
этап следующего хода, а постоянные цены новости № 34 сохраняются до конца
партии. Оставшиеся эффекты не подменяются заглушками.

Артефакты имеют `publishable: false`: это проверенный исполняемый черновик, но
не готовая к выпуску игра. Структуру входного снимка задаёт соседняя JSON
Schema, проверяемая Ajv в строгом режиме.

Сознательная перегенерация:

```bash
npm run generate:cards-money-trains-cargo-news
```

Проверка исходника, схемы, детерминизма и актуальности производного файла:

```bash
npm run verify:cards-money-trains-content
```

Сознательная сборка жизненного цикла после обновления входного снимка:

```bash
node games/cards-money-trains/tools/build-card-lifecycle.mjs
npm run compile:manifests -- --game cards-money-trains
```

Проверка детерминизма генератора и выполнение правил через обычный серверный
исполнитель:

```bash
node games/cards-money-trains/tools/build-card-lifecycle.mjs --check
node --experimental-strip-types --test \
  games/cards-money-trains/tools/card-lifecycle.test.mjs \
  games/cards-money-trains/tools/news-economic-effects.test.mjs \
  games/cards-money-trains/tools/news-progressive-tax.test.mjs \
  games/cards-money-trains/tools/news-scalar-effects.test.mjs
npm run compile:manifests -- --game cards-money-trains --check
```

## Описания стран из PDF

`import-country-descriptions.mjs` воспроизводимо извлекает 10 страниц из
`draft/trains/Описания стран Гвинеи 1 стол.pdf` системным `pdftotext`; новые
пакеты для этого не нужны. Производный
`authoring/fixtures/country-descriptions.intake.json` сохраняет:

- путь и контрольную сумму точной версии PDF;
- номер страницы и исходный порядок каждого описания;
- исходное название, строку терминалов и метки терминалов;
- текст с исходным переносом строк и копию, в которой мягкие переносы PDF
  соединены пробелами.

Артефакт имеет статус `review-draft` — «черновик для проверки» — и
`publishable: false`. Его номера терминалов остаются исходными метками
авторского документа: импортёр не объявляет их подтверждёнными
идентификаторами карты, не назначает странам `countryId`, не добавляет
координаты и не подключает содержимое к runtime-манифесту.

Сознательная перегенерация:

```bash
npm run generate:cards-money-trains-country-descriptions
```

Проверка выполняется общей дешёвой командой содержимого:

```bash
npm run verify:cards-money-trains-content
```

## Связанные страны и каталог

`build-country-content.mjs` соединяет проверенные описания с игровыми
объектами, не меняя исходный PDF-снимок. Генератор создаёт 10 стран с
устойчивыми идентификаторами, связывает с ними терминалы 1–23 и добавляет
описания в два доступных ведущему места:

- карточку выбранного терминала на карте;
- отдельный каталог стран, который позволяет открыть описание без выбора
  терминала.

Это уже исполняемая связь содержимого с картой, однако она остаётся
непубликационным игровым черновиком. Точные полигоны стран и областей ещё не
подготовлены, поэтому страна пока не является кликабельной геометрической
областью карты.

Сознательная перегенерация и узкая проверка:

```bash
node games/cards-money-trains/tools/build-country-content.mjs
node games/cards-money-trains/tools/build-country-content.mjs --check
node --experimental-strip-types --test \
  games/cards-money-trains/tools/country-content-linking.test.mjs
```

## Настройка нечётной партии

`build-session-setup.mjs` создаёт непубликационный, но исполнимый серверный
срез обычной настройки. Он использует только общие операции игрового движка:
создание объектов, выбор, воспроизводимое случайное упорядочивание, переход по
очереди и атомарное изменение состояния.

В срез входят:

- динамические команды вместо прежних четырёх записей с фиксированными
  идентификаторами;
- подтверждённые нечётные составы из 5, 7, 9 или 11 команд, где компаний-
  перевозчиков на одну больше, чем паровозных гильдий;
- 10 начальных монет, два резервных вагона перевозчика либо один резервный
  локомотив гильдии;
- неповторяющийся цвет из ограниченной контрастной палитры;
- воспроизводимая случайная очередь размещения;
- размещение ведущим на открытых терминалах технической начальной сети;
- ограничение не более двух локомотивов на терминале и переход к следующей
  команде только после размещения всей её техники.

Техническая сеть берётся из `annotations/initial-network.review.json`. Она
допустима для разработки и проверок, но остаётся непубликационной до
визуального подтверждения наложения автором. Чётные составы также намеренно
отклоняются: правило их соотношения ещё не подтверждено (`R-28`).

Генератор жизненного цикла карт больше не владеет моделью команд и не может
вернуть прежнюю фиксированную структуру. Рекомендуемый порядок сознательной
перегенерации:

```bash
node games/cards-money-trains/tools/build-card-lifecycle.mjs
npm run generate:cards-money-trains-session-setup
npm run compile:manifests -- --game cards-money-trains
```

Узкая проверка настройки:

```bash
npm run verify:cards-money-trains-session-setup
npm run compile:manifests -- --game cards-money-trains --check
```

## Старт первого хода и обязательное обслуживание

`build-operating-turn.mjs` соединяет завершённую расстановку, физические колоды
и первый обычный ход. Генератор остаётся локальным для этой игры и собирает
правила из существующих общих операций Mechanics: проверок состояния,
денежного перевода, изменения атрибутов объектов, ограниченной выборки и
журналируемого события.

Исполняемая граница включает:

- защищённый запуск обычной партии только после полной настройки и подготовки
  колод;
- явный пропуск новости первого хода без расходования карты или случайности;
- оплату по одной монете за каждый активный локомотив, вагон и удерживаемый
  груз;
- определение плательщика по владельцу объекта в серверном состоянии, а не по
  данным от интерфейса;
- атомарный отказ при повторной оплате, недоступном объекте или недостатке
  денег;
- завершение обслуживания только после обработки всех обязательных единиц;
- прямое завершение без списаний и без пообъектных действий при новости № 25;
- точный пересчёт прогрессивного налога новости № 14 при завершении
  обслуживания и атомарный отказ при недостатке денег;
- переход к фазе `market`;
- явное завершение рынка без сделок и без изменения денег, техники или
  рыночного состояния;
- завершение отчётности после пустого строительства, увеличение номера хода
  ровно на один и переход к новости следующего хода.

Покупка и продажа, реальное строительство и содержательная отчётность
намеренно не входят в генератор. Он задаёт только безопасные границы фаз,
которые позволяют не останавливать техническую многоходовую проверку. Поэтому
пакет остаётся непубликационным и сохраняет `runtimeReady: false`.

Сознательная последовательная перегенерация:

```bash
node games/cards-money-trains/tools/build-session-setup.mjs
node games/cards-money-trains/tools/build-card-lifecycle.mjs
node games/cards-money-trains/tools/build-operating-turn.mjs
npm run compile:manifests -- --game cards-money-trains
```

Три генератора композиционно идемпотентны: повторный запуск каждого после
остальных не должен менять результат. Узкая проверка границы:

```bash
node games/cards-money-trains/tools/build-session-setup.mjs --check
node games/cards-money-trains/tools/build-card-lifecycle.mjs --check
node games/cards-money-trains/tools/build-operating-turn.mjs --check
node --experimental-strip-types --test \
  games/cards-money-trains/tools/operating-turn-lifecycle.test.mjs \
  games/cards-money-trains/tools/news-progressive-tax.test.mjs
npm run compile:manifests -- --game cards-money-trains --check
```

## Порядок локомотивов и явный пропуск движения

`build-movement-order.mjs` реализует отдельную серверную границу после рынка и
выбора грузов. Минимальный переход из рынка без сделок уже соединён с грузовой
фазой, но полноценные рыночные операции и приоритетная последовательность
выбора грузов ещё не готовы.
Генератор не вычисляет порядок в интерфейсе и не добавляет предметную ветку в
общий сервер. Он собирает правило игры из существующих универсальных операций
выборки, многокритериального упорядочивания, сохранённой последовательности,
перемещения по объявленному графу, изменения состояния и журналирования.

В границу входят:

- выбор всех активных локомотивов с защитой от рассогласования их служебных
  признаков;
- порядок по положению справа налево, затем по деньгам владельца, затем по
  числу его активных локомотивов; полное равенство разрешается
  воспроизводимой случайностью;
- однократное сохранение полного порядка и текущего локомотива на весь этап;
- установка пяти единиц действия каждому участвующему локомотиву;
- перемещение только текущего локомотива по выбранной ведущим дороге
  технической сети: интерфейс передаёт дорогу, а идентификатор локомотива
  остаётся серверным;
- выбор дороги кликом непосредственно по карте либо через динамический
  доступный список текущих публичных дорог; оба пути отправляют только
  `{edgeId}` и не вычисляют локально допустимость движения;
- атомарная проверка открытости дороги и станций, смежности, вместимости,
  перенос прицепленных вагонов и списание одной единицы действия;
- при действующей новости № 22 — атомарный разовый сбор с владельца при
  первом успешном движении конкретного локомотива; недостаток денег отклоняет
  весь переход, но не запрещает ведущему явно пропустить локомотив;
- возможность выполнить несколько переходов тем же локомотивом до явного
  пропуска либо исчерпания его запаса действия;
- явный пропуск только сервером выбранного текущего локомотива, без
  передаваемого из интерфейса идентификатора;
- переход к следующему элементу сохранённого порядка, а после последнего
  пропуска — к отдельному этапу расчётов;
- отдельные записи игрового журнала о подготовленном порядке, каждом пропуске
  и завершении этапа движения.

Формирование состава, погрузка и доставка реализованы следующими отдельными
срезами. Обслуживание уже соединено с грузовым этапом явным завершением рынка
без сделок. Техническая стартовая сеть и вся обычная партия остаются
непубликационными до реализации настоящего рынка, приоритетного выбора грузов
и визуального подтверждения наложения автором.

Сознательная последовательная перегенерация:

```bash
node games/cards-money-trains/tools/build-session-setup.mjs
node games/cards-money-trains/tools/build-card-lifecycle.mjs
node games/cards-money-trains/tools/build-operating-turn.mjs
node games/cards-money-trains/tools/build-movement-order.mjs
npm run compile:manifests -- --game cards-money-trains
```

Узкая проверка:

```bash
node games/cards-money-trains/tools/build-movement-order.mjs --check
node --experimental-strip-types --test \
  games/cards-money-trains/tools/movement-order-lifecycle.test.mjs
npm run compile:manifests -- --game cards-money-trains --check
```

## Формирование состава текущего локомотива

`build-train-formation.mjs` добавляет к серверному этапу движения сохраняемый
выбор вагонов и атомарное групповое сцепление. «Атомарное» означает, что
операция либо полностью прицепляет все выбранные вагоны и списывает одну
единицу действия, либо не меняет партию вообще.

В срез входят:

- отдельные действия «отметить» и «снять отметку», принимающие только один
  скалярный `wagonId`; условного двойного переключения в одном плане нет;
- сохранение целевого локомотива на самом вагоне в публичном серверном
  состоянии, поэтому обновление страницы не теряет незавершённый выбор;
- текущий локомотив берётся только из серверного порядка движения и никогда
  не передаётся интерфейсом;
- проверка фазы, текущего и ещё не завершившего движение локомотива, наличия
  хотя бы одной единицы действия, типа и активности вагона, общей сети,
  станции, свободы и совместимости;
- подтверждение без параметров: сервер заново выбирает все отмеченные вагоны,
  сверяет полную группу и применяет универсальную ограниченную операцию
  `core.entities.each` в каноническом порядке идентификаторов;
- ровно одна единица действия за всю подтверждённую группу независимо от числа
  вагонов;
- очистка отметок после успешного сцепления и при явном пропуске текущего
  локомотива;
- одна точная запись игрового журнала с локомотивом, полным списком вагонов,
  их количеством, владельцем, стоимостью и номером хода;
- управление через карту кликом по вагону, видимый знак сохранённого выбора и
  те же действия в обычных доступных формах ведущего. Интерфейс не вычисляет
  допустимость: окончательное решение всегда принимает сервер.

## Погрузка, доставка и расчёты

`build-cargo-settlement.mjs` добавляет обычной партии погрузку выбранного
груза и отдельный этап расчётов после движения. Интерфейс передаёт только
идентификаторы публичных объектов; команды-владельцы, локомотив, маршрут,
выплаты и тариф берутся из подтверждённого состояния на сервере.

В срез входят:

- `cargo.load` с закрытыми параметрами `{wagonId, cargoId}`: активный пустой
  вагон должен принадлежать держателю груза, находиться на открытом исходном
  терминале и использовать ту же сеть;
- груз может остаться у логистической компании без погрузки; завершение этапа
  блокируется только незавершённым предложением карт, а не придуманной
  последовательностью команд;
- последний явный пропуск локомотива открывает этап `settlement`, сохраняя
  очистку незавершённого выбора состава;
- `settlement.cargo.deliver` принимает только `{wagonId}` и на сервере
  определяет груз, прицепленный локомотив и обе команды;
- банк сначала переводит стоимость груза с текущей надбавкой логистической
  компании, затем компания полностью платит гильдии по 2 монеты за ребро
  текущего кратчайшего открытого маршрута;
- недостаток средств после банковской выплаты отклоняет всю доставку вместе с
  переводами, журналом и изменением связей;
- успешная доставка журналирует полный расчёт, отцепляет вагон, освобождает
  его грузовой слот, помечает карту доставленной и снимает владение картой;
- отдельное завершение расчётов открывает строительство даже при грузе,
  доставку которого пришлось отклонить из-за недостатка средств.

Генератор не реализует рынок, очередность выбора грузов, правило одной
оставшейся карты или новости № 28/29. Эти ограничения остаются явными
границами следующей работы.

## Динамическое строительство

`build-construction-cycle.mjs` заменяет прежние действия, рассчитанные на
фиксированные идентификаторы четырёх команд, шестью параметризованными
действиями обычной партии. Ведущий может менять согласованные вклады любой из
4–12 команд, выбирать режим, строить несколько дорог и полустанков, а затем
отдельно завершить фазу.

Сервер сам определяет маршрут дороги с минимальным числом областей, стоимость
по две монеты за область и воспроизводимый случайный выбор при равенстве.
Вклады остаются только предварительными соглашениями: деньги списываются
атомарно вместе с созданием объекта и не списываются при отказе. Полустанок
стоит пять монет, проверяется вне областей обоих концов дороги и делит её на
две части. Обычные построенные объекты открываются в начале `N+2`.
Обе конечные станции строящейся дороги также закрываются до её открытия; если
несколько проектов используют одну станцию, она ждёт самого позднего срока,
а независимые причины закрытия не снимаются.

Новость № 26 предоставляет скидку до шести сегментов только первой успешно
построенной дороге текущего хода; полустанок и отклонённая команда скидку не
расходуют. Новость № 27 создаёт государственную дорогу `12 ↔ 22`, которая
открывается в `N+1`, либо безопасно завершается без изменения сети, если такая
дорога уже существует.

Пока автор не подтвердил наложение сети и точные замкнутые контуры областей,
генератор использует явно технические вертикальные области поверх
канонического поля `5079 × 3627`. Они позволяют исполнять и проверять весь
цикл, но не являются содержимым для публикации. Исходные дороги получают
согласованный с этими областями проверяемый план маршрута; после замены
геометрии игровые действия и серверные операции менять не потребуется.

```bash

node games/cards-money-trains/tools/build-construction-cycle.mjs --check

  games/cards-money-trains/tools/construction-cycle-lifecycle.test.mjs
```

Проверка покрывает идемпотентность генератора, произвольный состав 4–12
команд, атомарность денег и графа, неверную цену и недостаток средств,

## Методические паузы и финальная рефлексия

`build-facilitated-session.mjs` добавляет игровой методический сценарий —
управляемые ведущим остановки, во время которых участники обсуждают
происходящее и возвращаются к партии. Для каждой из двух пауз доступны только
три явных перехода: начать, отложить и завершить. Постоянный пропуск паузы и

4. К чему адаптироваться не удалось? Почему?
5. Как бы вы оценили результаты игры для вас и для других команд?

## Повторяемый технический цикл ходов

`build-operating-turn.mjs` также добавляет две закрытые команды ведущего без
параметров:

- `market.phase.finish` переводит рынок в грузовую фазу без покупки, продажи,
  изменения денег, техники или рыночного состояния;
- `reporting.phase.finish` доступна только после корректно завершённого
  строительства, фиксирует в журнале номер завершённого хода, увеличивает его
  ровно на один и открывает новостной этап следующего хода.

В результате техническая последовательность имеет вид:

```text
setup → news → maintenance → market → cargo → movement → settlement
  → construction → reporting → news
```

Метки уже выполненных действий текущего хода не очищаются принудительно:
после увеличения номера они естественно относятся к предыдущему ходу.
Временные эффекты новости сбрасываются только защищённым действием открытия
следующей новости или периода застоя. Постоянные цены, созданные объекты,
последовательность строительства и остальные долгоживущие данные сохраняются.

Для PM это означает: уже можно многократно прогонять техническую партию,
явно пропуская сделки на рынке и используя строительство на технических
областях. Это ещё не полная партия: остаются покупка и продажа с конечным
запасом рынка, предложение одной грузовой карты, новости № 19, 28 и 29,
проверка наложения начальной сети, реальные полигоны областей, правило
чётного состава команд и финальный сценарий с итоговым расчётом, долгами и
учебной рефлексией.

## Граница ответственности

Векторный результат относится только к этой игре и остаётся
`review-draft`. Сам инструмент извлечения не назначает
`regionId`/`countryId`, не соединяет линии в полигоны и не создаёт фрагмент
runtime-манифеста. Отдельный генератор уже назначает идентификаторы 10 стран,
связывает их описания с терминалами 1–23 и добавляет каталог. Однако точные
полигоны стран и областей по-прежнему требуют ручной проверки структуры
контуров и потому не готовы для публикации.

Исходный снимок описаний остаётся редакторским материалом и не становится
источником координат. Нормативной для текущего непубликационного игрового
черновика является только воспроизводимая связь, созданная
`build-country-content.mjs`; будущая геометрия должна сохранить эти
идентификаторы.
