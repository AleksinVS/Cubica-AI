## Назначение

Этот каталог — отдельная запускаемая тестовая игра с идентификатором
`cards-money-trains-mock`. Она позволяет развивать общий экран ведущего,
транспортное поле, строительство и восстановление сессии, не выдавая
вымышленные данные за решения автора.

**Всё содержимое здесь помечено MOCK и не предназначено для публикации.**
Нормативная игра остается в соседнем каталоге `games/cards-money-trains` и не
изменяется сборщиком тестовой копии.

## Что уже можно проверять

- связную сеть из шести терминалов и шести исходных дорог;
- три вымышленные области и стоимость дороги 2 монеты за сегмент области;
- серверное строительство дороги между двумя существующими узлами по
  минимальному числу областей; точная ломаная и версия геометрии сохраняются в
  сессии, а равные варианты используют воспроизводимую случайность;
- добавление полустанка по доле полной длины сохранённой дороги, включая
  ломаную;
- атомарное списание вкладов четырех команд без отрицательного баланса;
- mock-грузы, выплаты, новость и заранее известный контрольный сценарий;
- полный цикл `setup → news → maintenance → market → cargo → operations →
  construction → debrief → finished` без автоматического лимита ходов;
- скрытый воспроизводимый порядок двух колод: шесть новостей и двенадцать
  грузов, при этом наружу выходят только текущая карта и предложение;
- покупку и продажу техники только на рынке, без кредита и отрицательного
  баланса;
- загрузку груза, прицепление, ручное отцепление, движение и автоматическое
  отцепление после доставки;
- выбор фактически активного локомотива и существующей дороги прямо на карте
  или в обычной клавиатурной форме; интерфейс передает только идентификаторы
  выбранных объектов, а допустимость движения по-прежнему определяет сервер;
- расчет выплаты и тарифа `2 × число ребер кратчайшего открытого маршрута`;
- повторяемое строительство до отдельного завершения этапа;
- две методические паузы, финальную рефлексию и отдельные рейтинги перевозчиков
  и гильдий;
- общий экран ведущего и Phaser-проекцию публичного состояния.

Все эти правила используют общие декларативные операции Mechanics. Игровые
последовательности собираются в authoring-макросах, а исполняются как
типизированные атомарные планы. Специальных веток по идентификатору тестовой
игры в runtime или player-web нет.

## Почему это отдельная игра

Отдельный идентификатор не позволяет случайно запустить mock-сессию как
авторскую. Сервер загружает пакет обычным универсальным путем по gameId: в
ядре платформы нет проверки имени этой игры и нет специальных веток кода.

Тестовая копия наследует утвержденную оболочку нормативной игры во время
сборки и использует локальную копию разрешённой авторской карты как визуальную
подложку. Собственными остаются только синтетические аннотации, топология,
фикстуры и игровое содержимое. В сгенерированном манифесте сохраняется ссылка
на нормативный gameId и признак обязательной замены mock-содержимого перед
публикацией. Саму карту публиковать разрешено; запрет относится к тестовому
пакету с вымышленными правилами и данными.

## Как пересобрать пакет

Из корня репозитория:

```bash
node games/cards-money-trains-mock/tools/build-mock-package.mjs
node scripts/manifest-tools/compile-authoring-manifests.cjs --game cards-money-trains-mock
node scripts/manifest-tools/build-player-web-plugin-bundles.cjs --game cards-money-trains-mock
```

Первый шаг проверяет аннотацию и формирует authoring-исходники. Второй создает
исполняемые game/UI-манифесты. Третий собирает неизменяемый браузерный плагин.

## Как импортировать тестовое содержание

Новости, грузы, методические паузы, роли и инструкции хранятся отдельно от
механических настроек в `fixtures/mock-text-content.json`. Структуру этого
файла задает JSON Schema `fixtures/mock-text-content.schema.json`: это единый
машиночитаемый источник истины для формата импортируемого содержания.

Проверить файл, ничего не записывая:

```bash
node games/cards-money-trains-mock/tools/import-mock-text-content.mjs --check
```

Создать проверенную копию в `generated/mock-text-content.imported.json`:

```bash
node games/cards-money-trains-mock/tools/import-mock-text-content.mjs
```

Обычная сборка mock-пакета вызывает тот же модуль автоматически и берет
карточки, роли и инструкции только из отдельного входного файла. Если схема
или ссылки между игровыми данными некорректны, сборка завершается до записи
authoring-манифестов. Импортер записывает результат через временный файл и
атомарное переименование, поэтому ошибка не повреждает предыдущий результат.
Запись в каталог нормативной игры `games/cards-money-trains` запрещена самим
инструментом.

## Как заменить mock-данные ответом автора

1. Создать копию `annotations/map-annotation.template.json`.
2. Перенести подтвержденные узлы, дороги и области по инструкции
   `annotations/README.md`.
3. Получить проверенный фрагмент и наложение командой конвертера.
4. Передать автору наложение для ручной проверки.
5. Только после подтверждения заменить данные нормативной игры отдельным
   исполнительным изменением. Автоматического переноса из mock-пакета нет.
6. Заменить грузы, выплаты, новости и контрольный сценарий отдельными
   подтвержденными файлами.

## Контрольная полная партия

`fixtures/control-transcript.json` — короткий первый ход для диагностики
операций и строительства.

`fixtures/complete-session-transcript.json` — семиходовая приемочная партия из
88 подтвержденных действий. Она проходит все девять фаз, содержит два
контрольных перезапуска, семь безопасных отказов, две методические паузы,
отмену первого запроса завершения и окончательный ручной финал. Седьмой ход
доказывает, что число шесть является только длиной учебного примера, а не
автоматическим пределом партии.

## Граница готовности

Пакет считается полезным для разработки, когда узкие тесты проходят и
контрольный сценарий исполняется сервером. Он **никогда** не считается готовым
к продуктовой публикации: для этого требуется заменить данные авторскими и
принять нормативный пакет игры.
