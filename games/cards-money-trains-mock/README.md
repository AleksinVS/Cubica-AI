## Назначение

Этот каталог — отдельная запускаемая тестовая игра с идентификатором
`cards-money-trains-mock`. Она позволяет развивать общий экран ведущего,
транспортное поле, строительство и восстановление сессии, не выдавая
вымышленные данные за решения автора.

**Всё содержимое здесь помечено MOCK и не предназначено для публикации.**
Нормативная игра остается в соседнем каталоге `games/cards-money-trains` и не
изменяется сборщиком тестовой копии.

## Что уже можно проверять

- связную сеть из шести терминалов и шести исходных дорог;
- три вымышленные области и стоимость дороги 2 монеты за сегмент области;
- строительство дороги между двумя существующими узлами;
- добавление полустанка в середину существующей дороги;
- атомарное списание вкладов четырех команд без отрицательного баланса;
- mock-грузы, выплаты, новость и заранее известный контрольный сценарий;
- серверный цикл новости, атомарного обслуживания, движения по открытому ребру
  и доставки груза с двумя денежными расчетами;
- общий экран ведущего и Phaser-проекцию публичного состояния.

Контрольный груз уже проходит движение и выплату. Свободная погрузка,
прицепление произвольных вагонов, расчет кратчайшего пути и полные колоды
остаются последующими вертикальными срезами.

## Почему это отдельная игра

Отдельный идентификатор не позволяет случайно запустить mock-сессию как
авторскую. Сервер загружает пакет обычным универсальным путем по gameId: в
ядре платформы нет проверки имени этой игры и нет специальных веток кода.

Тестовая копия наследует утвержденную оболочку нормативной игры во время
сборки, но внедряет собственные аннотации, фикстуры и ресурс карты. В
сгенерированном манифесте сохраняется ссылка на нормативный gameId и признак
обязательной замены содержимого перед публикацией.

## Как пересобрать пакет

Из корня репозитория:

```bash
node games/cards-money-trains-mock/tools/build-mock-package.mjs
node scripts/manifest-tools/compile-authoring-manifests.cjs --game cards-money-trains-mock
node scripts/manifest-tools/build-player-web-plugin-bundles.cjs --game cards-money-trains-mock
```

Первый шаг проверяет аннотацию и формирует authoring-исходники. Второй создает
исполняемые game/UI-манифесты. Третий собирает неизменяемый браузерный плагин.

## Как заменить mock-данные ответом автора

1. Создать копию `annotations/map-annotation.template.json`.
2. Перенести подтвержденные узлы, дороги и области по инструкции
   `annotations/README.md`.
3. Получить проверенный фрагмент и наложение командой конвертера.
4. Передать автору наложение для ручной проверки.
5. Только после подтверждения заменить данные нормативной игры отдельным
   исполнительным изменением. Автоматического переноса из mock-пакета нет.
6. Заменить грузы, выплаты, новости и контрольный сценарий отдельными
   подтвержденными файлами.

## Граница готовности

Пакет считается полезным для разработки, когда узкие тесты проходят и
контрольный сценарий исполняется сервером. Он **никогда** не считается готовым
к продуктовой публикации: для этого требуется заменить данные авторскими и
принять нормативный пакет игры.
