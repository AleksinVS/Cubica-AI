## Назначение

Этот каталог содержит общий **schema-first конвейер** — процесс, в котором
машиночитаемая JSON Schema является единственным источником истины для формы
данных. Конвейер принимает ручную разметку изображения: узлы транспортной
сети, дороги, области и координатную систему. Он проверяет связи и геометрию,
а также строит SVG-наложение для визуальной сверки.

Канонический формат находится в
`docs/architecture/schemas/map-annotation.schema.json`. Нейтральные фикстуры
в `fixtures/` доказывают, что инструмент не зависит от конкретной игры.

## Граница ответственности

Общий модуль отвечает за:

- проверку JSON Schema;
- уникальность идентификаторов и целостность ссылок дорог;
- границы координат, замкнутость и отсутствие самопересечений областей;
- сверку контрольной суммы исходного изображения, если она указана;
- безопасное SVG-наложение с локальной ссылкой на фон;
- механическое преобразование подтверждённых данных по настройкам игры.

Модуль не распознаёт правила по пикселям. Стоимость строительства, типы
объектов, допустимые состояния и начальный номер последовательности передаёт
адаптер конкретной игры. Поэтому эти значения не могут незаметно стать
общеплатформенными правилами.

## Статусы проверки

- `review-draft` — промежуточная разметка. Она может не содержать областей,
  может использовать состояние `unknown`, а спорные записи отмечает через
  `confidence`, `reviewNote` и `reviewIssues`. Для неё доступно SVG-наложение,
  но создание фрагмента манифеста запрещено.
- `template`, `mock`, `author-confirmed` — строгие статусы. Области обязательны,
  у узлов и дорог должны быть известные runtime-состояния, а каждый узел
  должен находиться хотя бы в одной области.

SVG для `review-draft` показывает заметное предупреждение. Неизвестные дороги
и узлы выделяются янтарным пунктиром, чтобы их нельзя было принять за
подтверждённые данные.

## Подключение игры

Игра импортирует из `map-annotation.mjs` общие функции и хранит рядом с собой
только настройки преобразования. Тонкий адаптер обычно:

1. переэкспортирует `validateMapAnnotation` и
   `createMapAnnotationReviewOverlaySvg` под локальными именами;
2. вызывает `createTransportManifestFragment(annotation, options)` с
   правилами своей транспортной модели;
3. использует `runMapAnnotationCli` для одинаковой обработки аргументов и
   записи результатов.

Такой адаптер не является вторым конвейером: в нём нет собственной проверки
формата, геометрии или генерации SVG.

## Проверка

Сфокусированную проверку запускают отдельно от полного набора проекта:

```bash
npm run verify:map-annotations
```

Она проверяет нейтральный строгий пример, неполный `review-draft`, запрет его
публикации и основные семантические ошибки.
