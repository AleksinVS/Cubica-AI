## Назначение

Этот каталог хранит точные снимки внешних Agent Skills, которые рассматриваются
как источники практик. Кандидат может служить референсом или исходным материалом
для будущей адаптации, но сам не является проектным навыком Cubica.

## Почему кандидаты неактивны

Upstream-навык может вводить собственные правила планирования, согласования,
Git-операций, оркестрации или проверки. Прямое включение создало бы несколько
конкурирующих процессов. Поэтому среда агентов обнаруживает только канонические
навыки из `skills/`; для Codex они дополнительно доступны через символические
ссылки `.codex/skills/`.

`skill-candidates/` не входит в эти пути, не перечисляется в активном реестре и
не должен сканироваться агентом как источник исполняемых инструкций.

## Состав

Машиночитаемый [реестр](./registry.json) фиксирует источник, commit, лицензию,
состояние upstream и контрольные суммы каждого импортированного файла. Сейчас
сохранены четыре кандидата, использованные при ревью `C_ui-from-mockup`:

- `agent-skills/frontend-ui-engineering`;
- `anthropic-skills/frontend-design`;
- `openai-skills/frontend-skill`;
- `superpowers/brainstorming`.

Кандидат OpenAI сохранён из последнего commit, где он ещё присутствовал, и
помечен как удалённый из текущего upstream.

## Переход в активный каталог

Для активации нельзя копировать каталог в `skills/` без изменений. Нужно:

1. применить `external-skill-adapter`;
2. выделить одну узкую способность;
3. удалить конфликтующие правила;
4. сохранить проектную версию в `skills/`;
5. зарегистрировать происхождение и пройти проверки проекта.

## Обновление

Кандидаты не обновляются автоматически. Плановая проверка выполняется не чаще
одного раза в 30 дней, кроме прямого запроса пользователя или события
безопасности. Обновление заменяет снимок только после смыслового просмотра
изменений и пересчёта реестра.
