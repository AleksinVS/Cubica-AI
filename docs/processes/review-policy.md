# Политика ревью (черновик)

## Цель
Зафиксировать минимальные правила ревью (review — проверка изменений перед слиянием), чтобы:
- изменения были понятны новым участникам;
- архитектурные решения не расходились между сервисами/SDK.

## MVP-правила
- Один PR — одна осмысленная задача (ссылка на `docs/tasks/`).
- Для изменений архитектуры — ADR в `docs/architecture/adrs/`.
- Обязательна проверка ссылок в документации и актуальности `PROJECT_STRUCTURE.md` при изменениях структуры.
