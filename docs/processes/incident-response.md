# Реагирование на инциденты (черновик)

Инцидент (incident — событие, приводящее к нарушению работы системы) фиксируется для того, чтобы:
- восстановить сервис;
- собрать факты и причины;
- снизить вероятность повторения.

## Минимальный процесс
1. Описание симптомов и времени начала.
2. Временное восстановление (mitigation — снижение ущерба).
3. Причина (root cause — первопричина).
4. План действий: исправление, тесты, мониторинг.
