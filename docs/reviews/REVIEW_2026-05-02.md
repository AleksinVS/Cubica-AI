## 1. Общая оценка структуры проекта

Проект Cubica представляет собой монорепозиторий (monorepo) с вполне осмысленной топ-уровневой структурой: `apps/`, `services/`, `SDK/`, `packages/`, `games/`, `docs/`, `draft/`. Эта структура логична для платформенного проекта и позволяет разделять delivery-слой, backend runtime, SDK и игровой контент.

**Сильные стороны:**
- Четкая декомпозиция на канонические слои: `games/antarctica/` (контент), `services/runtime-api/` (backend), `apps/player-web/` (frontend), `packages/contracts/` (контракты).
- Наличие архитектурной документации: ADR, PROJECT_OVERVIEW, PROJECT_STRUCTURE, ROADMAP.
- `runtime-api` содержит детальные интеграционные тесты (3157 строк), что говорит о попытке держать opening flow под контролем.
- Использование npm workspaces для управления зависимостями в монорепозитории.

**Слабые стороны:**
- Значительный разрыв между декларируемой архитектурой и фактической реализацией. Например, `DEV_GUIDE.md` декларирует строгие модульные границы, но `index.ts` сервиса нарушает их, экспортируя внутренние детали.
- Переизбыток временных/черновых артефактов. Каталог `draft/` содержит 867 файлов (312 ad-hoc скриптов, 130+ PNG), `sandbox/` — 32 файла, `.tmp/` — 371 файл. Все это попало в git и увеличивает размер репозитория.
- Frontend-компонент `antarctica-player.tsx` (1673 строки) совмещает API-клиент, бизнес-логику, рендереры и панели — это делает его неподдерживаемым.
- Backend `runtime-api` жестко привязан к игре `antarctica`, что противоречит заявленной цели "единый deployable backend".
- Не все пакеты имеют рабочие скрипты `build`, `lint`, `test` — вместо этого стоят заглушки `echo "TODO..."`.

**Оценка:** проект находится в фазе активного MVP-delivery. Архитектурный каркас существует, но требует жесткой инженерной дисциплины, чтобы tech debt не превратил репозиторий в "болото".

## 3. Предложения по улучшению

### 3.1. Критичные (сделать первыми)

1. **Убрать `@next/swc-linux-x64-gnu` из `dependencies`** (`apps/player-web/package.json`).
2. **Исправить `useTransition`/`startTransition`** в `antarctica-player.tsx` — заменить на явный `isPending` через `useState`, или использовать `useActionState`.
3. **Добавить санитизацию для `dangerouslySetInnerHTML`** (например, `DOMPurify`) или явный комментарий о доверенности источника.
4. **Удалить из индекса Git** и добавить в `.gitignore`:
   - `.tmp/` (371 файл)
   - `draft/*.cjs`, `draft/*.png`, `draft/*.jpg` в корне `draft/`
   - `draft/**/node_modules/`, `draft/**/.next/`
   - `sandbox/nanobanana-output/`
   - `*.log` в корне
5. **Усилить `.gitignore`** — добавить правила для draft-артефактов и sandbox-generated assets.
6. **Удалить дубликаты ассетов**: `3.png` (дубль `i34.png`), 7 файлов с "копия", 5 `.design.json` из `drafts/`.
7. **Убрать `.gitkeep`** из всех модулей `services/runtime-api/src/modules/*`, где уже есть код.
