# Ежемесячный обзор технического долга (Debt Review)

Документ описывает минимальный процесс регулярного обзора (review — проверка и обсуждение) технического долга.
Источник правды по долгу: `docs/legacy/debt-log.csv` и `docs/legacy/stubs-register.md`.

## Оглавление
- [Цель](#цель)
- [Регламент](#регламент)
- [Шаблон записи](#шаблон-записи)

## Цель
- Поддерживать список заглушек (stub — временная подмена/упрощение) актуальным.
- Снимать или перепланировать долг до того, как он станет блокером разработки.

## Регламент
1. Раз в месяц проверить все записи со статусом `active`.
2. Обновить `last_reviewed_at` в `docs/legacy/debt-log.csv`.
3. Для высоких рисков (`risk_level=high`) подтвердить план снятия и владельца.
4. Если заглушка удалена — обновить `docs/legacy/stubs-register.md` и статус в CSV.

## Шаблон записи
Для заметок к обзору используйте формат:
- Дата, участники
- Ключевые изменения/решения
- Какие `LEGACY-*` закрыты / какие требуют пересмотра сроков
