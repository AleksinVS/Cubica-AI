## Описание
Этот эпик описывает первый этап разработки игровой платформы: проектирование структуры JSON‑манифеста игр, схем (schema) для этих манифестов и базовых принципов хранения и обработки данных в LLM-first архитектуре.

В соответствии с `PROJECT_OVERVIEW.md` и ADR‑001:
- фронтенд и SDK строятся по шаблону MVP (Model–View–Presenter);
- игровой слой строится по принципу LLM-first: LLM выступает игровым «движком» и принимает на вход JSON‑манифест + текущее состояние игры;
- структура JSON‑манифеста может отличаться у разных игр, поэтому вместе с манифестом всегда хранится его схема;
- Model и View не знают друг о друге и взаимодействуют только через Presenter;
- данные и логика хранятся совместно, а описание UI — отдельно (в других разделах манифеста или отдельных файлах);
- манифест содержит системный промт для LLM, выступающей в роли игрового «движка».

Задача этого эпика — описать _рамки_ (формат, разделы, обязательные поля и принципы) для манифестов и их схем так, чтобы на их основе можно было уже реализовывать конкретные игры (например, «Antarctica») без пересмотра базовой архитектуры.

## Результат-для-пользователя-и-разработчика

После выполнения эпика:
- для пользователя: появится возможность запускать разные игры/сценарии поверх одного и того же плеера, так как все игры описываются манифестами совместимого формата;
- для разработчика:
  - будет задокументирован единый формат JSON‑манифеста и его схем;
  - будет понятно, куда класть данные и логику, а где — описывать UI;
  - будет описан протокол взаимодействия Presenter ↔ LLM (какие поля манифеста и состояния передаются в LLM и какой формат ответов ожидается).
