## Why

PM (2026-07-20): «спланируй и реализуй переворачивание игровых карточек».
Переворот карточки‑решения (лицо = выбор, оборот = последствие) — сигнатурная
механика исходной «Антарктиды», которой сейчас нет в плеере. Архитектура под неё
уже заложена (фасет `face`, `backText`, удержание разрешённой карточки рантаймом),
но не соединена в рабочее поведение. Решение зафиксировано в ADR-094.

## Scope

В границах:
- Двусторонний рендер `cardComponent` (лицо/оборот) и анимированный переворот.
- Проводка в игре Antarctica: `face→back` при разрешении, view‑правило, привязка
  `backText`, перекомпиляция манифестов, оформление оборота.
- Нейтральная тестовая фикстура, доказывающая общность возможности.

Вне границ:
- Изменения turn-flow рантайма (не требуются — карточка удерживается «на месте»).
- Новые публичные поля контракта/схемы (переиспользуем `backText` + `visualState`).
- Переворот нарративных карточек без `backText`.

## Классификация (общее vs игровое)

- **Общее (платформа):** двусторонний рендер и переворот `cardComponent`,
  структурный 3D‑CSS (scoped модуль компонента), контрактная семантика
  `backText` + `visualState=resolved`. Доказывается нейтральной фикстурой.
- **Игровое (Antarctica):** контент `backText`, шаг `face→back` при разрешении,
  view‑правило `face.back → visualState=resolved`, привязка `backText` в
  UI‑шаблоне, визуальное оформление граней в `games/antarctica/assets/styles`.

## Найденное состояние (разведка)

- Рантайм: `game.card.resolve` (action `opening.card.*`) ставит
  `selection=selected`, `resolution=resolved`, применяет метрики, держит
  `timeline.canAdvance`; экран не сменяется автоматически (кнопка «Далее»).
  → Разрешённая карточка остаётся на экране и может быть перевёрнута.
- Контент: у всех карточек‑решений задан `backText`
  (`games/antarctica/game.manifest.json`, `authoring/game.authoring.json`).
- Модель объекта: `objectModels.antarctica.card.facets.face = {front, back}`
  объявлена, но не используется; view‑правила есть только у `availability.*`.
- Контракт: `GameUiCardComponentProps.backText` и `.visualState` (incl.
  `resolved`) уже существуют; UI‑manifest schema допускает `backText`.
- Рендерер: `apps/player-web/src/components/manifest/card-component.tsx` не
  привязывает `backText`, не создаёт двусторонней структуры, не переворачивает.
- Стили: `games/antarctica/assets/styles/antarctica.css` содержит мёртвые
  `.default-game-card` / `.flipped` (перенос из черновика).
