## Understanding

Владелец закрыл два продуктовых вопроса из TSK-20260704 (см. LEGACY-0041):

1. **Политика apply-гейта.** Смягчение до «нет НОВЫХ ошибок» ОТКЛОНЕНО. Причины:
   ломает инвариант «манифест всегда компилируется» (на нём стоит весь
   preview-first конвейер), требует хрупкого diff-а множеств диагностик по
   устойчивому ключу, допускает каскадное маскирование новых ошибок за старыми,
   рассинхронизирует apply/save/preview. Вместо смягчения — усиление UX «почини
   сначала» (**Вариант А**): понятная причина блокировки + быстрый путь к
   исправлению.
2. **Язык UI редактора — русский** (single-locale). Сейчас chrome смешан:
   доменные подписи из макета русские, действия-инструмента английские
   (`Save`, `Undo`, `Preview`, `Compile`, `Inspect`, `Collapse`, …). Приводим к
   единому русскому языку интерфейса и выносим строки в locale-модуль, чтобы не
   было дрейфа (единый источник строк, как в player-web).

«chrome» (обрамляющая оболочка инструмента) — верхняя панель, боковые панели,
вкладки, строка состояния, надписи кнопок и подписи для скринридеров
(aria-label); в отличие от «контента» — данных игры, которые редактирует автор.

## Scope

### Часть 1 — Вариант А (apply-UX «почини сначала»)

1. **Тултипы на заблокированных действиях.** Кнопки `Save`/`Compile`/`Preview`,
   заблокированные из-за блокирующих диагностик, получают `title` с числом
   блокирующих ошибок и подсказкой открыть «Проверки» / перейти к первой ошибке
   (переиспользуется существующий `handleNavigateToFirstError`,
   `workspaceCheckCounts`). Инвариант гейта НЕ меняется.
2. **Массовый детерминированный quick-fix недостающего `_label`.** Новый
   движковый билдер в `packages/editor-engine/src/entity-operations.ts`
   (`buildFillEntityLabelChangeSet`) — проставляет производный по умолчанию
   `_label` (гуманизация id) для tree-visible сущности без `_label`
   (game-agnostic; диагностика `schema.ts` «must define a non-empty _label»).
   В панели «Проверки» — новый quick-fix `fill-label` на строке и групповое
   действие «Исправить все (N)» для одинаковых детерминированно-исправимых
   диагностик. Всё идёт через единую точку apply
   (EditorChangeSet → risk → dry-run → validation → undo).

### Часть 2 — Русификация chrome

3. **Locale-модуль editor-web** `apps/editor-web/src/lib/locale/ru.ts` (+ `index.ts`)
   — типизированный объект строк chrome, единый источник (SSOT) с богатыми
   комментариями по ключам; framework-free.
4. **Миграция компонентов** (~24 файла): видимый текст и `aria-label`/`title`
   английских действий → строки из locale-модуля (русский). Доменные подписи из
   макета, уже русские, сохраняются.
5. **Синхронное обновление тестов.** e2e-спека `editor-session-preview.spec.ts` и
   юнит-тесты, ассертящие английские подписи (`Preview`, `Collapse`, `Inspect`,
   `Timeline`, `Reset to start`, …), обновляются на русские в один заход.

## Non-Goals

- Полноценный переключатель локалей / вторая локаль (single-locale русский).
- Локализация контента игр (у контента свой язык, идёт из манифеста).
- Изменение политики гейта или инварианта валидности (явно отклонено).
- Концепты §9.7–9.8 (отдельный TSK-20260707).

- Заблокированные `Save`/`Compile`/`Preview` объясняют причину (счётчик
  блокирующих ошибок) и дают переход к первой ошибке; инвариант гейта не изменён.

  соответствующие блокирующие ошибки исчезают, манифест снова валиден.
- Весь chrome редактора на русском (видимый текст и aria-label); строки — из
  locale-модуля; захардкоженных английских подписей действий не осталось.
- `verify:editor-engine` зелёный; editor-web build + unit зелёные; e2e prod 8/8.
