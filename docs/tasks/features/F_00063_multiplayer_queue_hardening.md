# FEATURE: Укрепление очереди событий мультиплеера

## Контекст и цели

- [ ] Обеспечить надёжную обработку действий нескольких игроков в одной сессии за счёт явной версионности состояния и последовательности событий.
- [ ] Снизить риск рассинхронизации между состоянием в БД и журналом событий, а также подготовить систему к последующему масштабированию Router.

## Объём

- In scope:
  - Уточнение ADR-011 по очереди событий и версионности состояния (`state_version`, `last_event_sequence`).
  - Введение правил обработки зависших и ошибочных событий (`pending` / `processing` / `failed`).
  - Добавление в протокол обновлений для клиентов информации о версии состояния и последнем событии.
  - Подготовка каркаса реализации очереди событий в сервисе Router (типы и интерфейсы).
- Out of scope:
  - Реализация полной интеграции Router с реальной БД и LLM.
  - Нагрузочные оптимизации и сложные стратегии масштабирования (отдельные воркеры, внешние очереди и т.п.).

## Задачи

- [ ] Уточнить ADR-011:
  - [ ] Добавить описание полей `state_version` и `last_event_sequence` в `game_sessions`.
  - [ ] Расширить схему `session_events` полями `sequence`, `attempts`, `error_code`.
  - [ ] Описать правила обработки зависших событий и повторных попыток.
  - [ ] Уточнить формат обновлений для клиентов (версии и события).
- [ ] Обновить ExecPlan:
  - [ ] Поддерживать в актуальном состоянии `CP_00060_multiplayer_architecture_hardening.yaml` по мере выполнения шагов.
- [ ] Подготовить каркас реализации очереди событий в Router:
  - [ ] Добавить модуль с типами и интерфейсами работы с очередью событий.
  - [ ] Добавить базовые тесты, проверяющие наличие и корректность этих интерфейсов.
- [ ] Синхронизировать документацию:
  - [ ] Обновить `docs/architecture/PROJECT_ARCHITECTURE.md`, чтобы он ссылался на уточнённый ADR-011.
  - [ ] Проверить, что ROADMAP и данная фича отражают фактическое состояние работ.

## Acceptance Criteria

- [ ] ADR-011 отражает:
  - [ ] Версионность состояния и связь с событиями.
  - [ ] Расширенную схему таблицы `session_events`.
  - [ ] Правила обработки зависших и ошибочных событий.
  - [ ] Формат обновлений для клиентов с версиями.
- [ ] ExecPlan `CP_00060` ведётся и остаётся самодостаточным.
- [ ] В `services/router/` добавлен модуль с типами и интерфейсами очереди событий и базовыми тестами.
- [ ] `docs/architecture/PROJECT_ARCHITECTURE.md` и `docs/tasks/ROADMAP.md` согласованы с текущим состоянием мультиплеерной архитектуры.

## Definition of Done

- [ ] ADR-011 обновлён и согласован с архитектурой Router.
- [ ] ExecPlan `CP_00060` переведён в статус `done` с заполненным разделом `Outcomes & Retrospective`.

- [ ] Обновлены ROADMAP и текущая фича с актуальными чек-листами.
