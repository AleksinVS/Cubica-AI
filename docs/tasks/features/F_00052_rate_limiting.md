# F_00052: Rate Limiting & Budget Control

- **Эпик**: [E_0050](../epics/E_0050_observability_and_quality.md)
- **Статус**: Planned
- **Приоритет**: Medium

## Описание
Реализация механизмов ограничения нагрузки и контроля расходов на LLM.

## Проблема
Отсутствие защиты от злоупотреблений может привести к:
1.  DDoS атакам через игровые сессии (истощение ресурсов Engine).
2.  Неконтролируемым финансовым расходам на API провайдеров LLM.

## Задачи
1.  Реализовать Rate Limiter в Router Service (алгоритм Token Bucket или Leaky Bucket).
2.  Внедрить лимиты:
    *   Запросов в минуту на сессию/пользователя.
    *   Токенов LLM на сессию.
3.  Добавить конфигурацию лимитов в манифест игры.
4.  Реализовать Graceful Degradation (корректный возврат 429 статуса).

## Acceptance Criteria
- [ ] Система блокирует избыточные запросы с кодом 429.
- [ ] Превышение бюджета токенов останавливает сессию с понятным сообщением.
