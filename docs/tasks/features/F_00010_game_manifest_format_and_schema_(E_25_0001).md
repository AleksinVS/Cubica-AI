# FEATURE: Базовый формат JSON-манифеста игр и схем

## Оглавление
- [Цели](#цели)
- [Scope](#scope)
- [Acceptance-Criteria](#acceptance-criteria)
- [Definition-of-Done](#definition-of-done)

## Цели

- [x] Определить базовую структуру JSON‑манифеста игр в духе LLM-first архитектуры: какие разделы и поля считаются обязательными/рекомендуемыми.
- [x] Определить подход к описанию схем (schema) для манифестов и привязке конкретного манифеста игры к его схеме.
- [x] Зафиксировать принципы разделения данных/логики и описания UI внутри манифеста.

## Scope

In scope:
- Описание разделов манифеста на уровне концепций (без детального проектирования всех полей):
  - данные и логика игры (состояния, сущности, правила, ограничения), удобные для LLM;
  - системный промт LLM (роль, ограничения, формат ответа);
  - описание UI (экраны, области, виджеты) отдельным разделом/поддокументом;
  - метаданные (версии, ссылки на схемы, идентификаторы игр).
- Правила различий между играми:
  - что считается «ядром» формата (общее для всех игр);
  - какие части могут варьироваться и как это отражается в схемах.
- Принципы хранения:
  - где лежат манифесты (например, `games/*` или отдельный сервис);
  - где и как хранятся схемы (например, `docs/architecture/schemas/*` или в отдельном репозитории).

Out of scope:
- Детальное описание всех полей для конкретных игр (например, «Antarctica»).
- Реализация конкретных бэкенд‑сервисов или плееров — это отдельные задачи.

## Acceptance-Criteria

- [x] Подготовлен текстовый документ (или серия документов) в `docs/architecture/schemas/` или `games/*`, описывающий базовый формат манифеста и схем.
- [x] В документе явно указано:
  - какие разделы манифеста обязательны для всех игр;
  - какие разделы могут отличаться и как для них описываются схемы;
  - как в манифесте хранится системный промт LLM.
- [x] Определены базовые инварианты:
  - Model и View для конкретной игры используют один манифест и одну/несколько схем;
  - данные и логика игры хранятся совместно, описание UI — отдельно;
  - Presenter получает от LLM чётко заданный формат ответов (обновление Model и подсказки по изменению View).
