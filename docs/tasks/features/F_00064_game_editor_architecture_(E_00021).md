# FEATURE: Game Editor — Architecture

## Контекст и цели

Описание архитектуры Game Editor — графического редактора для создания обучающих игр на платформе Cubica. Документ определяет структуру компонентов, их взаимодействие и технические решения.

## Задачи

- [ ] Описать общую архитектуру Game Editor
- [ ] Определить структуру компонентов UI (боковая панель чата, поле редактора)
- [ ] Описать модель данных редактора (состояние, манифесты, сессия)
- [ ] Определить протокол взаимодействия с LLM-агентом
- [ ] Описать режимы работы редактора (диалог, сценарий, мокапы, прототип, отладка)
- [ ] Интегрировать архитектуру с существующими компонентами платформы (viewers, manifests)

## Ключевые компоненты

### 1. Интерфейс редактора
- **Chat Panel** — боковая панель для диалога с ИИ-агентом
- **Editor Canvas** — основное поле редактора (контекстно меняется в зависимости от этапа)
- **Toolbar** — панель инструментов для управления редактором

### 2. Режимы редактора
- **Dialog Mode** — подготовительный диалог (этап A)
- **Scenario Mode** — редактор сценария с flow-диаграммой (этап B)
- **Mockup Mode** — редактор мокапов (этап C)
- **Prototype Mode** — сборка и отображение прототипа (этап D)
- **Debug Mode** — запуск и отладка игры (этап E)

### 3. Слой данных
- **Editor State** — состояние редактора (текущий режим, выбранные элементы)
- **Game Manifest** — манифест создаваемой игры
- **Session** — сессия редактирования (история изменений, контекст LLM)

### 4. Интеграция с LLM
- **Agent Context** — контекст для ИИ-агента (схемы, правила, текущее состояние)
- **Skills** — навыки агента для каждого этапа редактирования
- **Prompts** — системные промты для направления агента
