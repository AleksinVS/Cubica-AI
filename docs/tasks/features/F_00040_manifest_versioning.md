# F_00040: Manifest Versioning Strategy

- **Эпик**: [E_0010](../epics/E_0010_game_manifest_architecture.md)
- **Статус**: done
- **Приоритет**: Critical

## Описание
Разработка и внедрение стратегии версионирования игровых манифестов для обеспечения обратной совместимости и управляемой эволюции платформы.

## Проблема
Текущая схема не предусматривает поля версии. Любое изменение формата `game.json` приведет к неработоспособности всех существующих игр.

## Задачи
- [x] Добавить поле `meta.schema_version` в спецификацию манифеста.
- [x] Добавить поле `meta.min_engine_version` для указания требований к движку.
- [x] Реализовать проверку версии при загрузке игры в Game Engine (Design in ADR-008).
- [x] Создать **ADR-008: Manifest Versioning Strategy**.

## Acceptance Criteria
- [x] Манифест содержит информацию о версии схемы (schema_version).
- [x] Манифест содержит информацию о минимальной версии движка (min_engine_version).
- [x] Движок отклоняет игры с неподдерживаемой версией схемы с понятной ошибкой (Specified in ADR-008).
- [x] Описан процесс миграции старых манифестов (в ADR-008).
- [x] Создан ADR-008.
