- [x] Передана начальная сеть поверх общей карты того же размера.

- [x] Из AI воспроизводимо извлечены 23 векторных центра терминалов и 981
  кандидат линии; калибровка к карте проверена с максимальной ошибкой
  `0,697` пикселя, создано отдельное непубликационное SVG-наложение.
- [ ] Автор подтвердил четыре вопроса по типам специальных точек, доступности

  контрольное наложение.
- [ ] Импортированы и проверены грузы, новости и страны без изменения исходных файлов.
- [x] Связь новости № 12 уточнена: закрывается только `3 ↔ 3,14`, соседние дороги не затрагиваются.

- [x] Карта зарегистрирована; PM подтвердил, что использование согласовано с автором и правообладателем и не несет правового риска.
- [x] На тестовых данных готовы команды, фазы, деньги, граф, роль ведущего и Phaser-host; это не закрывает авторские входы выше.

## 5. Перед GSR-032

- [x] Переданы исходные файлы; сеть извлечена в `review-draft`, AI разобран и
  откалиброван по 23 терминалам. Получены 981 непубликационный кандидат;
  смысловая классификация и замыкание областей ещё не выполнены.
- [x] Подтверждено: дорога допустима через любую область, полустанок — на любой существующей дороге, путь выбирает система по минимуму областей.
- [x] PM принял ADR-081: сервер получает только концы, строит минимальную по
  непрерывным региональным сегментам траекторию и воспроизводимо случайно
  выбирает равноценный вариант. Общий контракт, нейтральная приёмка и
  test-only подключение выполнены; нормативная игра по-прежнему ждёт
  подтверждённых контуров областей.
- [x] Определено измерение отступа полустанка: точка не может лежать ни в одной области любого конца исходной дороги.
- [x] Определён точный момент открытия обычного строительства — начало `N+2`; все дороги двусторонние.
- [x] Контрольный mock-сценарий подтверждает атомарное списание только полной распределенной стоимости.
- [x] Общий lifecycle и mock-срез проверяют точную границу: строительство хода
  `N` закрыто весь `N+1` и открывается в начале `N+2`.
- [x] В mock-сценарии есть груз, для которого новая сеть меняет маршрут или доступность.
- [x] Контрольная mock-фаза строит полустанок и дорогу до отдельного завершения;
  оба порядка строительства проходят одинаковый общий lifecycle без жёстко
  заданного идентификатора (`LEGACY-0056` закрыт).
- [x] Дорога выбирается двумя узлами на карте или в доступной форме; отдельный

## 7. Полная тестовая партия GSR-035

Этот раздел относится только к отдельной игре `cards-money-trains-mock` и не
снимает содержательные блокеры нормативной версии.

- [x] Все `metric.transfer` в исходниках, генераторе и скомпилированном
  манифесте используют `from.scope`/`to.scope` и `onInsufficient: "fail"`;
  старых полей `kind`/`insufficientFunds` нет.
- [x] Генератор mock-пакета выполнен, runtime/UI-манифесты заново скомпилированы,
  браузерный плагин пересобран; проверка рассинхронизации проходит.
- [x] Runtime реализует и серверными проверками покрывает фазы `setup`, `news`,
  `maintenance`, `market`, `cargo`, `operations`, `construction`, `debrief`,
  `finished`.
- [ ] Полный ручной проход ведущего через все фазы интерфейса ещё не принят:
  браузерная проверка покрывает видимый жизненный цикл, а точный 88-шаговый
  сценарий выполняется как серверная интеграция.
- [x] Общие `deck shuffle/draw` доказаны нейтрально; mock-колоды имеют не менее
  шести новостей и двенадцати грузов, скрывают будущее и воспроизводятся по seed.
- [x] Рынок проверяет покупку, продажу, владельца, фазу и недостаток денег.
- [x] Общие `load/attach/detach` доказаны нейтрально; mock проверяет свободный
  вагон, узел отправления, совместимость, единицу действия и автоснятие при
  доставке.
- [x] Общий shortest-route settlement — расчет и проведение платежа по
  кратчайшему маршруту — доказан нейтрально; mock показывает маршрут и состав
  тарифа до/после закрытия или строительства.
- [x] Основной контрольный сценарий позволяет выполнить несколько строительств
  до отдельного завершения; неполная оплата не оставляет частичного результата,
  оба порядка строительства проверены до автоматического открытия в `N+2`.
- [x] Строительные действия не используют зашитые идентификаторы: узлы,
  существующая дорога и положение полустанка выбираются через карту либо DOM,
  а подтверждение дороги требует актуального серверного расчёта.
- [x] Общий ranking compute — расчет мест по объявленной формуле — доказан
  нейтрально; mock показывает отдельного победителя/равенство для каждого типа.
- [ ] Техническое действие паузы и финальная рефлексия существуют, но ход 3 и
  ход 5 пока запускают один id, а импортированный вопрос не управляет экраном
  (`LEGACY-0059`).
- [x] Шестиходовой контрольный replay включает успешный путь и безопасные отказы;
  после шестого хода возможен следующий ход без автоматического завершения.
- [x] Запрос завершения можно отменить и подтвердить; после `finished` игровые
  мутации отклоняются.
- [x] PostgreSQL-восстановление проверено после строительства и после открытия
  карты; состояние и будущий порядок колод не меняются.
- [ ] Browser-сценарий проверяет создание сессии, старт с клавиатуры,
  перезагрузку и видимый финал, но длинную середину проводит прямыми
  API-запросами. Полный ручной путь через все фазы интерфейса не доказан.
- [x] UI и данные заметно помечены `MOCK`; gameId отличается от нормативного,
  автоматического копирования в `cards-money-trains` нет. Пользовательская
  выгрузка пока не реализована и отдельно отложена как `LEGACY-0053` и
  `LEGACY-0054`.

## 8. Интерфейс «карта — основа»

- [x] PM принял общий декларативный режим; ADR-080 имеет статус `Accepted` и
  синхронизирован с общей архитектурой.
- [x] Право использования, изменения и публикации текущей карты подтверждено;
  web-производная и происхождение зарегистрированы.
- [x] UI JSON Schema и authoring-схема принимают `layout_mode: "map-first"` и
  ограниченный `workspaceSlot`, отклоняют неизвестную зону и отсутствие или
  дублирование `board`.
- [x] Manifest TypeScript-контракты, Presenter и routing поддерживают новый
  режим без отдельной ручной формы, расходящейся со схемой.
- [x] Нейтральная фикстура доказывает общий режим без названий, ассетов и
  правил «Карт, денег, поездов».
- [x] Общий renderer заполняет рабочую область картой и размещает status,
  primary/context panels, action tray, floating controls и overlay по
  семантическим зонам, а не по игровым CSS-классам.
- [x] `interactiveBoardSurface` в map-first режиме не резервирует постоянную
  колонку; независимые DOM-действия открываются поверх карты и работают при
  ошибке Phaser.
- [x] Mock UI использует разрешённую авторскую карту только как визуальный фон,
  сохраняет крупную маркировку `MOCK` и не переносит нормативную сеть или
  правила.
- [x] Нормативный UI использует `board-guinea-optimized` и каноническую
  плоскость `5079 × 3627`; черновые узлы совмещены с ней в отдельном
  контрольном наложении.
- [ ] Подтверждённые узлы и контуры областей включены в runtime-проекцию;
  `review-draft` не считается выполнением этого пункта.
- [x] Камера имеет pan/zoom, границы, «Показать всю карту» и полноэкранный
  режим; повторное монтирование не накапливает обработчики.
- [x] На `1920 × 1080`, `1440 × 900` и `1024 × 768` canvas совпадает с рабочей
  областью с допуском 2 CSS-пикселя, а документ не имеет обычной вертикальной
  прокрутки.
- [x] На узком контрольном размере панели превращаются в нижние листы;
  критические действия остаются доступны с клавиатуры.
- [ ] Визуальная приёмка по
  `map-first-interface-specification.md` выполнена; `LEGACY-0062` закрыт только
  после фактического доказательства.
