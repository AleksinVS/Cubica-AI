# Исполнительный пакет «Карты, деньги, поезда»

Папка содержит постоянные рабочие спецификации корневой задачи `TSK-20260711-cards-money-trains-game`. Общий план и статус остаются в TSK; эти документы дают исполнителю подробные входные данные и доказательства приемки, но не создают параллельный план.

## Оглавление

- [Состав пакета](#состав-пакета)
- [Порядок чтения](#порядок-чтения)
- [Границы](#границы)

## Состав пакета

- `product-specification.md` — понятная продуктовая спецификация полной первой версии.
- `author-materials-request.md` — самодостаточный запрос автору: срочные материалы, простые шаблоны и способ передачи транспортной сети изображением.
- `content-intake-contract.md` — формат и критерии получения недостающего авторского содержимого.
- `requirements-traceability.md` — связь требований с источниками, архитектурой, игровыми срезами и проверками.
- `implementation-readiness-checklist.md` — контроль входов и доказательств перед каждым срезом и завершением.
- `trial-run-guide.md` — пошаговый запуск mock-игры с PostgreSQL, остановка,
  возобновление и диагностика для PM и ведущего.

## Порядок чтения

1. Анализ первичных материалов в `docs/reviews/2026-07-11-cards-money-trains-source-analysis.md`.
2. `product-specification.md` для продуктового результата.
3. `docs/architecture/cards-money-trains-platform-design.md` для границ платформы и игры.
4. GSR-030–033 — нормативные игровые срезы, GSR-035 — полностью проходимая
   тестовая партия до получения авторского содержимого.
5. Корневой TSK для очередности исполнения и текущего статуса.

## Границы

При конфликте правил приоритет имеет указанный PM главный DOCX, затем явные решения PM. При конфликте архитектурных трактовок приоритет имеют принятые ADR. Файлы `draft/` не читаются runtime — серверным исполняющим слоем — и не являются исполнимым контрактом.

До получения недостающих материалов разработка использует отдельную запускаемую тестовую копию `games/cards-money-trains-mock/`. Её вымышленные данные не являются содержимым авторской игры и не переносятся в нормативный пакет `games/cards-money-trains/` без проверки и подтверждения автора.
