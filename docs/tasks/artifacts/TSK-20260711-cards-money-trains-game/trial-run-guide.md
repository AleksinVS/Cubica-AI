# Пробный запуск «Карты, деньги, поезда» на тестовых данных

Руководство позволяет ведущему запустить локальную тестовую версию игры,
провести несколько ходов, остановить программы и затем продолжить ту же
сессию. Тестовая версия использует вымышленные команды, сеть, карточки и цены.
Она проверяет работу цифровой игры, но не подтверждает авторские правила или
баланс.

## 1. Что будет запущено

Нужны три локальных процесса:

1. **PostgreSQL** — база данных, в которой постоянно хранится сессия.
2. **Runtime API** — сервер правил: он проверяет действия, меняет деньги,
   транспортную сеть и фазы, затем сохраняет результат.
3. **Player Web** — браузерный общий экран ведущего с обычными панелями и
   интерактивной картой Phaser.

База слушает только локальный адрес компьютера. Интерфейс открывается по адресу:

```text
http://127.0.0.1:3000/?gameId=cards-money-trains-mock
```

На экране должна быть заметна маркировка `MOCK — только разработка`. Если её
нет, не проводите пробную партию: вероятно, открыта другая игра.

## 2. Предварительная проверка

Откройте терминал в корне репозитория `/home/abc/projects/Cubica-AI` и
выполните:

```bash
node --version
docker version
```

Нужен Node.js версии не ниже `22.7.0` и работающий Docker. Если зависимости
репозитория еще не установлены, один раз выполните:

```bash
npm install
```

Дальше понадобятся два открытых терминала в корне репозитория. База данных
работает в фоне и отдельного терминала не требует.

## 3. Первый запуск

### 3.1. Создать постоянное хранилище PostgreSQL

Эта команда создает именованный том Docker — постоянную область данных, которая
не исчезает после обычной остановки контейнера:

```bash
docker volume create cubica-cmt-pgdata
```

Запустите PostgreSQL 17:

```bash
docker run -d \
  --name cubica-cmt-postgres \
  -e POSTGRES_USER=cubica \
  -e POSTGRES_PASSWORD=cubica_local_only \
  -e POSTGRES_DB=cubica \
  -p 127.0.0.1:55432:5432 \
  --mount type=volume,src=cubica-cmt-pgdata,dst=/var/lib/postgresql/data \
  --health-cmd "pg_isready -U cubica -d cubica" \
  --health-interval 2s \
  --health-timeout 5s \
  --health-retries 20 \
  postgres:17
```

Подождите, пока база станет готова:

```bash
docker inspect --format '{{.State.Health.Status}}' cubica-cmt-postgres
```

Нормальный ответ — `healthy`. Если пока показано `starting`, подождите несколько
секунд и повторите проверку.

### 3.2. Запустить сервер правил

В первом терминале задайте локальную базу, примените схему хранения и запустите
сервер:

```bash
export SESSION_STORE=postgresql
export DATABASE_URL='postgresql://cubica:cubica_local_only@127.0.0.1:55432/cubica'
export PORT=3001
npm run migrate:sessions --workspace @cubica/runtime-api
npm run dev --workspace @cubica/runtime-api
```

Терминал должен остаться открытым. Сообщение
`runtime-api listening on :3001` означает, что сервер запущен.

### 3.3. Запустить общий экран ведущего

Во втором терминале выполните:

```bash
RUNTIME_API_URL=http://127.0.0.1:3001 \
PORT=3000 \
NEXT_IGNORE_INCORRECT_LOCKFILE=1 \
npm run dev --workspace @cubica/player-web
```

Терминал также должен остаться открытым. После сообщения о готовности откройте:

```text
http://127.0.0.1:3000/?gameId=cards-money-trains-mock
```

При первом открытии браузер автоматически создает сессию и сохраняет её
идентификатор в локальном хранилище сайта. Отдельно создавать сессию командой не
нужно.

## 4. Проведение пробной партии

Перед началом убедитесь в трех признаках:

- заголовок — «Карты, деньги, поезда»;
- видна маркировка `MOCK — только разработка`;
- показаны тестовая транспортная карта и кнопка подтверждения команд.

Дальше ведущий использует кнопки группы текущей фазы. Строительство уже
использует фактические узлы и дороги из текущей сессии; выбранные на карте
значения появляются в обычной форме. Для груза и техники полный динамический
путь ещё не завершён (`LEGACY-0058`): интерфейс может показать уже
использованное или недоступное действие. Сервер не применит неверную операцию,
но сообщение может быть техническим; это известное ограничение, а не ошибка
ведущего. В тестовом цикле последовательно проходят настройка, новость,
обслуживание, рынок, грузы, операции, строительство и итоги хода. Игра не
заканчивается по номеру хода: завершение запрашивает и отдельно подтверждает
ведущий.

Для пробной приемки полезно проверить:

- начало игры клавишей `Enter`, а не только мышью;
- один успешный рыночный платеж и один отказ из-за недостатка денег;
- загрузку, сцепление, движение и доставку груза;
- несколько строительств до отдельного завершения фазы; разрешены и проверены
  оба порядка — «дорога → полустанок» и «полустанок → дорога»;
- для дороги — выбрать две станции, нажать «Рассчитать маршрут», сверить
  зелёную временную линию и цену, затем подтвердить строительство; после
  изменения станции прежний расчёт должен исчезнуть;
- обновление страницы в середине хода;
- остановку и возобновление по следующему разделу;
- запрос завершения, его отмену и последующее подтверждение.

## 5. Остановка без потери сессии

1. В терминале Player Web нажмите `Ctrl+C`.
2. В терминале Runtime API нажмите `Ctrl+C` и дождитесь сообщения об остановке.
3. Остановите базу:

```bash
docker stop cubica-cmt-postgres
```

Не удаляйте контейнер и том `cubica-cmt-pgdata`. Обычная остановка сохраняет
всю сессию.

## 6. Возобновление сохраненной сессии

Важно: до возобновления не пересобирайте и не заменяйте игровой пакет. Текущая
сессия хранит идентификатор игры, но еще не закрепляет неизменяемую версию её
правил. Этот разрыв зарегистрирован как `LEGACY-0055`.

Запустите базу:

```bash
docker start cubica-cmt-postgres
```

Убедитесь, что она готова:

```bash
docker inspect --format '{{.State.Health.Status}}' cubica-cmt-postgres
```

Затем снова выполните команды разделов 3.2 и 3.3. Откройте тот же адрес в том
же браузере и в том же профиле. Player Web прочитает сохраненный идентификатор
сессии и запросит её актуальное состояние из PostgreSQL.

Если открыть игру в другом браузере, другом профиле или приватном окне,
автоматически будет создана другая сессия.

## 7. Как начать отдельную новую сессию

Самый простой безопасный способ — открыть игру в новом приватном окне браузера.
Существующая сессия при этом остается в базе.

Для нового запуска в текущем профиле можно открыть инструменты разработчика
браузера, перейти в `Console` и выполнить:

```javascript
localStorage.removeItem("cubica-cards-money-trains-mock-session-id")
```

После обновления страницы Player Web создаст новую сессию. Команда удаляет
только локальную ссылку браузера и не стирает старую запись из PostgreSQL.

## 8. Диагностика

### База не запускается

Проверьте состояние и последние сообщения:

```bash
docker ps -a --filter name=cubica-cmt-postgres
docker logs --tail 100 cubica-cmt-postgres
```

Если Docker сообщает, что имя уже занято, контейнер уже создан. Используйте
`docker start cubica-cmt-postgres`, а не повторяйте `docker run`.

### Runtime API не готов

Проверьте отдельно сам процесс и его соединение с базой:

```bash
curl -fsS http://127.0.0.1:3001/health
curl -fsS http://127.0.0.1:3001/readiness
```

Первый запрос подтверждает, что процесс жив. Второй подтверждает, что доступно
постоянное хранилище. Если первый проходит, а второй нет, проверьте состояние
PostgreSQL и значение `DATABASE_URL` в терминале Runtime API.

### Игра не загружается

Проверьте доступность именно тестового пакета:

```bash
curl -fsS http://127.0.0.1:3001/games/cards-money-trains-mock/readiness
curl -fsS http://127.0.0.1:3001/games/cards-money-trains-mock/player-content
```

Если Runtime API отвечает, а браузер показывает ошибку, проверьте второй
терминал: Player Web должен быть запущен с
`RUNTIME_API_URL=http://127.0.0.1:3001`.

### Порты уже заняты

Для пробного запуска можно выбрать, например, `3101` для Runtime API и `3100`
для Player Web. Во всех командах согласованно замените порты:

```bash
export PORT=3101
# запуск Runtime API

RUNTIME_API_URL=http://127.0.0.1:3101 PORT=3100 \
NEXT_IGNORE_INCORRECT_LOCKFILE=1 \
npm run dev --workspace @cubica/player-web
```

Тогда адрес игры будет
`http://127.0.0.1:3100/?gameId=cards-money-trains-mock`.

### После перезапуска открылась новая партия

Проверьте следующие условия:

- используется тот же браузерный профиль;
- локальные данные сайта `127.0.0.1:3000` не очищались;
- запущен контейнер `cubica-cmt-postgres` с томом `cubica-cmt-pgdata`;
- Runtime API запущен с `SESSION_STORE=postgresql`, а не с временным
  `in-memory`;
- игровой пакет не менялся между остановкой и возобновлением.

## 9. Полное удаление тестовых данных

Следующие команды необратимо удаляют все локальные тестовые сессии из этого
контейнера. Выполняйте их только когда продолжение партий больше не нужно:

```bash
docker stop cubica-cmt-postgres
docker rm cubica-cmt-postgres
docker volume rm cubica-cmt-pgdata
```

После этого следующий запуск снова начинается с раздела 3.1.

## 10. Известные ограничения

- Это тестовый пакет с вымышленным содержимым, а не цифровое издание авторской
  игры.
- Версия игрового пакета пока не закреплена за сессией (`LEGACY-0055`), поэтому
  файлы игры нельзя менять между частями одной пробной партии.
- Технические события сохраняются, но полноценный пользовательский журнал
  ходов и выгрузка отложены как `LEGACY-0053`.
- Итоговый ИИ-анализ и учебная рефлексия отложены как `LEGACY-0054`. В будущем
  это будет отдельная производная от фактического журнала: она не изменит счет
  или состояние, не будет психологически оценивать участников и должна будет
  показывать, где факт, а где интерпретация.
- Строительные действия уже имеют динамический выбор и актуальный server
  preview. Для грузов, техники и полного прохода остальных фаз динамическое
  отсечение ещё неполно; сервер безопасно отклоняет неверное действие, но
  ведущему нужно учитывать текст отказа (`LEGACY-0058`).
- Методические вопросы, роли и инструкции уже импортируются как данные, но пока
  не выбираются и не показываются по состоянию сессии (`LEGACY-0059`).
- Построенные в ходе `N` объекты остаются закрыты весь `N+1` и автоматически
  открываются в начале `N+2`; независимое закрытие новостью при этом сохраняется.
- До исправления `LEGACY-0057` проверяйте движение на стартовых локомотивах:
  купленный локомотив пока не получает единицы действия на следующем ходу.
- Сессия автоматически возобновляется только в том же браузерном профиле.
  Не очищайте данные сайта; безопасный способ повторного подключения после
  потери localStorage отложен как `LEGACY-0060`.
