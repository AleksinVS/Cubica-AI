# FEATURE: Game Editor — Prototype Builder

## Контекст и цели

Этап создания прототипа игры. После того как мокап создан и утверждён, LLM-агент выполняет вёрстку экрана игры и отображает собранную (сверстанную) страницу в поле редактора. Пользователь может выбрать элементы на странице и указать агенту, как их нужно изменить.

Прототип — это интерактивная HTML-страница, которая показывает, как будет выглядеть и работать экран игры.

- [ ] Реализовать автоматическую вёрстку из мокапа
- [ ] Отобразить сверстанную страницу в редакторе
- [ ] Добавить возможность итеративных изменений через команды агенту

## Объём

**In scope:**
- Генерация HTML/CSS из JSON-мокапа (через LLM-агента)
- Отображение сверстанной страницы в iframe редактора
- Выделение элементов на сверстанной странице
- Итеративные изменения через команды агенту
- Синхронизация изменений с мокапом

**Out of scope:**
- Полноценный редактор кода HTML/CSS
- Сложная логика JavaScript (только базовая интерактивность)
- Экспорт готового кода (это отдельная фича)

## Задачи

- [ ] Реализовать генерацию HTML/CSS
  - [ ] Промт для LLM-агента: преобразование мокапа в вёрстку
  - [ ] Шаблон базовой HTML-страницы
  - [ ] Генерация CSS-стилей из описания мокапа

- [ ] Реализовать отображение прототипа
  - [ ] Компонент iframe для рендеринга страницы
  - [ ] Изоляция стилей (sandbox)
  - [ ] Поддержка responsive-режима (разные размеры экрана)

- [ ] Реализовать выделение элементов
  - [ ] Инъекция скрипта для отслеживания кликов в iframe
  - [ ] Подсветка элемента при наведении
  - [ ] Передача информации о выделенном элементе в редактор

- [ ] Реализовать итеративные изменения
  - [ ] Inline-поле ввода (как в Mockup Editor)
  - [ ] Формирование контекста: текущий код + выделенный элемент
  - [ ] Применение изменений от LLM
  - [ ] Обновление iframe

- [ ] Синхронизация с мокапом
  - [ ] Обновление JSON-мокапа при изменениях в прототипе
  - [ ] Регенерация прототипа при изменениях в мокапе

## Acceptance Criteria

- [ ] LLM-агент генерирует HTML/CSS из мокапа
- [ ] Сверстанная страница отображается в редакторе
- [ ] Пользователь может выделять элементы на странице
- [ ] Пользователь может запросить изменения через inline-поле
- [ ] Изменения применяются к прототипу в реальном времени
