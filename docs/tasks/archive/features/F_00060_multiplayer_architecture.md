# F_00060: Multiplayer Architecture

- **Эпик**: [E_0030](../epics/E_0030_backend_architecture_design.md)
- **Статус**: done
- **Приоритет**: Low (Phase 2)

## Описание
Проектирование и реализация поддержки многопользовательских игр.

## Проблема
Текущая архитектура рассчитана на Single Player. Нет механизмов для синхронизации действий нескольких игроков в одной сессии, разрешения конфликтов одновременных ходов и широковещательной рассылки обновлений UI.

## Задачи
1.  [x] Создать **ADR-011: Multiplayer Architecture** (Free-form с очередью событий).
2.  [x] Определить модель взаимодействия: **Free-form (Async)** с Event Queue.
3.  [x] Спроектировать расширение структуры стейта (`world`, `players`, `secret`).
4.  [x] Определить механизм бродкастинга через View Adapters (WebSocket Gateway).
5.  [x] Разработать стратегию разрешения конфликтов: **Last-Write-Wins** (Optimistic Locking).
6.  [x] Спроектировать таблицу `session_events` для очереди действий.

## Acceptance Criteria
- [x] Создан ADR-011 с описанием архитектуры мультиплеера.
- [x] Определена модель взаимодействия (Free-form с Event Queue).
- [x] Спроектирована структура стейта для поддержки нескольких игроков.
- [x] Определен механизм бродкастинга обновлений через View Adapters.
- [x] Выбрана стратегия разрешения конфликтов (Last-Write-Wins).
