# F_00062: Redis Integration

- **Эпик**: [E_0030](../epics/E_0030_backend_architecture_design.md)
- **Статус**: Planned
- **Приоритет**: Low (Phase 2)

## Описание
Интеграция Redis для масштабирования слоя хранения сессий.

## Проблема
Прямая работа с PostgreSQL (пессимистичные блокировки, чтение полного JSON) станет узким местом при росте нагрузки (High Load).

## Задачи
1.  Обновить **ADR-005**, детализировав стратегию использования Redis.
2.  Реализовать **Read-Through Cache** для сессий.
3.  Заменить блокировки БД (`SELECT FOR UPDATE`) на распределенные блокировки (Redis Lock / Redlock).
4.  Реализовать асинхронную запись в БД (Write-Behind) для снижения задержки (опционально, с оценкой рисков).

## Acceptance Criteria
- [ ] Время чтения сессии (Get Session) не зависит от нагрузки на БД.
- [ ] Блокировки работают корректно в распределенной среде (несколько инстансов Router).
