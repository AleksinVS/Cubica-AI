# FEATURE: Текстовые якоря и разделение логического и UI-манифестов

## Оглавление

- [Цели](#цели)
- [Scope](#scope)
- [Acceptance-Criteria](#acceptance-criteria)
- [Definition-of-Done](#definition-of-done)

## Цели

- [x] Зафиксировать на уровне архитектуры и схем, что смысловое текстовое содержимое игр хранится в файловых описаниях (правила, сценарии, методические материалы), а манифест является их структурированной проекцией.
- [x] Определить механизм текстовых якорей в описаниях (`<!-- anchor: ... -->` и Markdown‑ID) и формат ссылок `source_ref` в логическом манифесте.
- [x] Разделить логический манифест и UI‑манифесты: в логическом хранятся структура, состояние и действия, в UI‑манифестах — экраны, компоненты, связи с UI‑библиотекой и макеты.
- [x] Ввести сущность «макет» (изображение + JSON‑описание дизайна) в UI‑манифест и связать её с экранами/компонентами.
- [x] Продемонстрировать работу подхода на сценарии «Antarctica» (логический манифест + UI‑манифест + хотя бы один макет).

## Scope

In scope:
- Обновление архитектурной документации:
  - `docs/architecture/adrs/013-manifest-text-anchors-and-ui-split.md`;
  - разделы про манифесты в `PROJECT_ARCHITECTURE.md`;
  - `docs/architecture/schemas/manifest-structure.md` и `ui-schema-concept.md`.
- Обновление/расширение JSON Schema:
  - логический манифест (`manifest.v1.schema.json` или обновлённый `game-manifest.schema.json`) с поддержкой `source_ref`;
  - отдельный UI‑манифест (`ui-manifest.schema.json`) с разделами `screens` и `layouts`.
- Создание минимального tooling:
  - скрипт генерации/обновления логического манифеста из текстовых описаний (парсинг якорей);
  - базовая валидация манифеста по обновлённым схемам.
- Адаптация одной игры (Antarctica) под новый формат:
  - логический манифест с `source_ref`;
  - UI‑манифест с примерами экранов и хотя бы одним макетом.

Out of scope:
- Полный перенос всех существующих игр на новый формат.
- Полноценный визуальный редактор UI‑манифестов и макетов.

## Acceptance-Criteria

- [x] ADR‑013 обновлён и отражает финальное архитектурное решение, согласован с владельцем репозитория.
- [x] `manifest-structure.md` описывает:
  - текстовые источники и якоря;
  - разделение логического и UI‑манифестов;
  - сущность `source_ref` и ключевые инварианты.
- [x] В репозитории присутствует:
  - схема логического манифеста с `source_ref` и ссылками на текстовые файлы;
  - схема UI‑манифеста с разделами `screens` и `layouts`.
- [x] Реализован скрипт, который:
  - читает один или несколько текстовых файлов с описаниями сценария;
  - извлекает `anchor`‑комментарии;
  - обновляет хотя бы один пример логического манифеста (для Antarctica), заполняя `resolved` для полей с `source_ref`.
- [x] Сценарий «Antarctica» имеет:
  - отдельный логический манифест;
  - отдельный UI‑манифест с минимум одним макетом;
  - оба файла проходят валидацию по своим схемам.
