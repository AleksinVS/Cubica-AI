# F_00050: Testing Strategy for LLM Games

- **Эпик**: [E_0050](../epics/E_0050_observability_and_quality.md)
- **Статус**: Planned
- **Приоритет**: High

## Описание
Разработка стратегии и инструментария для тестирования игр, использующих LLM. Основной вызов — тестирование недетерминированной логики.

## Проблема
Традиционные Unit-тесты не подходят для LLM-based логики. Нет понимания, как гарантировать, что изменение промпта не сломает игру, и как проводить регрессионное тестирование.

## Задачи
1.  Создать документ `docs/architecture/testing-strategy.md`.
2.  Разработать подход к **Snapshot Testing** для диалогов (сравнение семантической близости ответов).
3.  Реализовать механизм **Mocking/Replay** для LLM API (запись эталонных ответов для детерминированного прогона тестов).
4.  Интегрировать тесты валидации манифестов в CI/CD.

## Acceptance Criteria
- [ ] Документирована стратегия тестирования.
- [ ] В проекте есть примеры тестов для Script-части (Unit) и LLM-части (Integration/Replay).
