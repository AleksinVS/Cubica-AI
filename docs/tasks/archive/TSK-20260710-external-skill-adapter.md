# TSK-20260710-external-skill-adapter: Адаптер внешних навыков

## Status

done

## Parent

none

## Why

ADR-069 принимает управляемую установку внешних навыков. Нужен проектный навык, который дешево и воспроизводимо проверяет отдельный upstream `SKILL.md` против компактного профиля Cubica, материализует совместимую версию и доказывает ее поведение без чтения всего репозитория.

## Scope

Входит в работу:

- определить JSON Schema и компактный профиль совместимости внешних навыков;
- добавить пополняемую память ошибок, исправлений и практик адаптации;
- добавить реестр зафиксированных upstream-версий и владения способностями;
- создать проектный навык адаптации с детерминированными скриптами анализа, сборки пакета и проверки результата;
- реализовать кэш по содержимому и версии политики;
- автоматизировать проверку upstream не чаще одного раза в 30 дней с ручным и security-исключениями;
- добавить сценарии проверки включения, полномочий, артефактов и конфликтов;
- провести пилотную адаптацию небольшого набора специализированных навыков из `agent-skills` и `superpowers`.

Не входит в работу:

- прямая установка полных внешних плагинов в активный профиль Cubica;
- автоматическое обновление upstream без проверки;
- перенос внешних мета-навыков и сквозных оркестраторов;
- изменение правил `$cubica`, `TSK-*` или архитектурных полномочий ADR-068;
- обязательное чтение всего репозитория моделью при обычной адаптации; точечное чтение необходимых источников разрешено.

## Acceptance

- Обычная адаптация начинает с одного внешнего навыка, минимального среза профиля и памяти, детерминированных находок и при обновлении измененных фрагментов.
- Агент может точечно читать необходимые проектные источники; причина и повторно полезный вывод фиксируются для следующих адаптаций.
- Неизменный ключ содержимого и версий профиля/памяти повторно использует кэш без модельного анализа.
- Плановая проверка upstream не выполняется чаще одного раза в 30 дней и не применяет обновление автоматически.
- Активный адаптированный навык не содержит конфликтующих требований по планированию, архитектурным полномочиям, Git-операциям, субагентам и включению `$cubica`.
- Реестр фиксирует upstream SHA, хэш, лицензию, статус, способность, версию политики и версию адаптера.
- Для одной способности по умолчанию существует один активный основной навык.
- Структурные, статические и поведенческие проверки блокируют несовместимый выпуск.
- `$cubica` остается только явно включаемым, остальные допущенные навыки могут включаться автоматически.
