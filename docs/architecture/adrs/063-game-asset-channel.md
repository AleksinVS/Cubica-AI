## Поправка 2026-07-19 (лимиты → рекомендации; тип css)

Эта датированная поправка изменяет исходное решение; текст ниже сохранён как
исторический, актуальные правила по перечисленным пунктам — здесь.

- **Жёсткие лимиты заменены рекомендациями без конкретных цифр.** Решение PM
  (2026-07-19): числовые лимиты исходного решения (файл ≤ 512 КБ, игра ≤ 4 МБ,
  ≤ 64 ассетов) больше не являются нормативной границей — объективно оценить её
  сейчас нельзя. Нормативная рекомендация: готовить ассеты экономно (сжатие,
  `webp` где уместно, без дублей и неиспользуемых файлов). CI больше не
  отказывает по размеру/количеству: соответствующие проверки понижены до
  **не валящих предупреждений** (`scripts/ci/validate-game-assets.js`,
  константы `ADVISORY_FILE_BYTES`/`ADVISORY_GAME_BYTES` — тюнингуемая эвристика,
  не контрактный лимит). Ограничение `maxItems: 64` снято из
  `game-assets.schema.json`.
- **Паттерн имён файлов остаётся нормативным.** Регулярное выражение `file` в
  схеме и связанные проверки (существование, отсутствие `..`/выход за каталог,
  отсутствие файлов-сирот, уникальность id) сохраняются как жёсткие отказы.
- **Допустимые типы ассетов расширены типом `css`** — см. ADR-091 (канал
  game-owned стилей). CSS-ассеты регистрируются в отдельной секции
  `stylesheets` реестра (`StylesheetAssetEntry`), делят пространство имён id и
  форму ссылки `asset:<id>` с изображениями и проверяются дополнительным
  правилом: ссылки на изображения внутри CSS — только токены `asset:<id>`
  (`validateCssSource`). Санитизация SVG и защитные заголовки при раздаче
  остаются без изменений.

## Как понято решение

Играм нужны собственные статические ассеты (изображения для Phaser-сцен и
DOM-экранов), но сегодня единственный работающий способ — класть файлы в
`apps/player-web/public/images/`, то есть в платформенный слой. Решается
вопрос: где живут ассеты игры, как они описываются, как доставляются клиенту
и как к ним обращаются сцена плагина и UI-манифест — без game-specific кода
в платформе.

## Контекст

- Файлы игры «Антарктида» (фоны, иконки) лежат в `apps/player-web/public/`
  и адресуются абсолютными URL `/images/...` из UI-манифестов — существующий
  незарегистрированный дрейф чистоты (правило 10 CLAUDE.md); оформлен как
  долг LEGACY-0023.
- ADR-062 ввёл Phaser-канал; его фикстура (`conveyor-mini`) сознательно
  ограничена примитивами. Продуктовые игры класса (первый потребитель —
  `rail-tycoon-mini`) требуют спрайтов.
- Игры реализуются целиком агентами: формат ассетов должен быть создаваем
  агентом как текст (не бинарь), с проверяемой CI безопасностью и
  зафиксированным происхождением/лицензией.
- Прецедент доставки игровых артефактов уже есть: публикуемые бандлы
  плагинов раздаются runtime-api контент-адресуемо
  (`/published-plugin-bundles/{gameId}/{pluginId}/{sha256}.mjs`) из
  `games/<id>/published/` с проверками путей.

## Решение

1. **Ассеты принадлежат игре.** Файлы живут в `games/<gameId>/assets/` и
   описываются декларативным реестром `games/<gameId>/assets/assets.json`
   по новой схеме `docs/architecture/schemas/game-assets.schema.json`
   (JSON Schema — SSOT, ADR-025/056; TS-контракты генерируются). Каждая
   запись: стабильный `id`, относительный путь `file`, `kind` (MVP — только
   `"image"`) и обязательное происхождение `origin`
   (`authored-in-repo` | `third-party` с полями `license` и `source`).
2. **Доставка — контент-адресуемая, по образцу бандлов плагинов.**
   runtime-api отдаёт индекс `GET /game-assets/{gameId}/index.json`
   (карта id → URL) и файлы
   `GET /game-assets/{gameId}/{assetId}/{sha256}.{ext}` с иммутабельным
   кэшированием; несовпадение хэша, id или расширения — 404. Файлы читаются
   только по путям из реестра внутри каталога ассетов игры.
3. **Обращение — только по id, никогда по пути.** Phaser-сцена получает в
   контексте резолвер `assets: GameAssetResolver` (`url(assetId)`, `ids()`;
   аддитивное расширение `PhaserSceneContext` ADR-062). UI-манифест ссылается
   на ассеты строковой формой `asset:<assetId>` в документированных
   image-свойствах; рендерер резолвит форму через индекс, оставаясь
   game-agnostic (ADR-055); неизвестный id — компонент без изображения,
   fail closed.
4. **SVG — основной агенто-писаемый формат.** Разрешённые форматы: png,
   jpg, webp, svg. Для svg нормативная санитизация в CI (запрет скриптов,
   обработчиков событий, `foreignObject`, внешних ссылок) плюс защитные
   заголовки при раздаче (`nosniff`, CSP без исполнения). Лимиты: файл
   ≤ 512 КБ, игра ≤ 4 МБ, ≤ 64 ассетов. *(Заменено поправкой 2026-07-19:
   лимиты — рекомендации без цифр; см. секцию поправки выше.)*
5. **Валидация — платформенный CI-скрипт** (`scripts/ci/validate-game-assets.js`
   в контуре `verify:canonical`): схема реестра, существование файлов,
   уникальность id, расширения, лимиты, санитизация svg, отсутствие
   файлов-сирот. Аудио и другие типы — вне MVP (расширение `kind` — новым
   решением; тип `css` добавлен ADR-091, см. поправку выше — размерные лимиты
   теперь не валящие рекомендации).

Нормативные детали (формы URL, заголовки, регэкспы, контракт резолвера,
правила санитизации) — `docs/architecture/game-asset-channel-design.md` §3.

## Отклонённые альтернативы

- **Продолжать класть файлы игр в `apps/player-web/public/`** — нарушает
  правило 10 (файлы игры в платформенном слое), не масштабируется, нет
  происхождения/лицензий; существующее размещение «Антарктиды» — долг
  LEGACY-0023, мигрируется отдельной задачей.
- **Раздача каталога ассетов «как есть», без реестра** — нет декларативного
  контракта (анти-паттерн правила 12), нет контроля происхождения, лимитов и
  санитизации; шире поверхность path traversal.
- **Ассеты внутри npm-пакета плагина** — конфликтует с
  `dependenciesPolicy: "platform-only"` (ADR-037) и смешивает код с данными;
  ассеты нужны и DOM-экранам без плагина.
- **Base64/data-URI в манифестах** — раздувает манифесты и проекции, ломает
  кэширование, нечитаемо в ревью.
- **Внешнее хранилище/CDN уже сейчас** — преждевременно: репозиторий — SSOT
  контента (модульный монолит, локальный файловый репозиторий); вынос — по
  operational-границам ADR-017, URL-схема к этому готова.

## Последствия

- Платформенная чистота: у игр появляется законное место для ассетов;
  новые игры не добавляют файлов в `apps/player-web`.
- Агентная разработка: SVG-ассеты пишутся агентом как текст и проходят
  детерминированную CI-проверку.
- Кэширование: контент-адресуемые URL дают безопасный
  `immutable`-кэш; смена файла меняет URL автоматически.
- Появляется небольшой рантайм-расход: вычисление SHA-256 файлов
  (кэшируется в памяти по mtime).
- Риск SVG как активного контента закрыт двумя рубежами: CI-санитизация
  коммитимых файлов + защитные заголовки при раздаче.
- `PhaserSceneContext` расширяется аддитивным полем `assets` — координация с
  владельцем контракта (трек flow-simulation) обязательна.
- Миграция картинок «Антарктиды» на канал — отдельная последующая задача;
  до неё LEGACY-0023 остаётся активным.
