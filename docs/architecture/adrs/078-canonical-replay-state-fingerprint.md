## 1. Контекст

Replay — повтор записанной последовательности действий от одинакового
начального состояния. Он должен доказать, что деньги, объекты, сеть, скрытые
колоды и результат игры совпали. При этом runtime сохраняет реальные отметки
времени для журнала ведущего, поэтому два последовательных запуска не могут
иметь буквально одинаковые JSON-снимки.

Удаление всех полей с распространёнными именами `at` или `lastUpdatedAt`
опасно: такое поле может быть частью игровых данных. Нужен единый платформенный
отпечаток с точным и версионированным правилом исключений.

## 2. Решение

1. Runtime предоставляет платформенную функцию канонического отпечатка полного
   серверного игрового состояния. Клиентская проекция не используется, потому
   что она намеренно скрывает будущий порядок колод и другие секретные данные.
2. Версия `cubica-replay-state-v1` рекурсивно сортирует ключи объектов обычным
   порядком JavaScript без языковой локали, сохраняет порядок массивов и
   сериализует JSON-примитивы стандартным `JSON.stringify`.
3. Из состояния исключаются только два точных пути:
   `public.log[*].at` и `runtime.lastUpdatedAt`. Сами поля остаются в сохранённой
   сессии и видимом журнале; канонизация не изменяет исходный объект.
4. Все одноимённые поля вне этих путей, скрытые колоды, состояние генератора
   случайных чисел и остальные игровые значения участвуют в сравнении.
5. Не-JSON значения, не конечные числа и циклические ссылки отклоняются явной
   ошибкой, а не нормализуются молча.
6. Результат имеет форму
   `cubica-replay-state-v1:sha256:<64 шестнадцатеричных символа>`. Изменение
   алгоритма или списка исключений требует новой версии политики.
7. Оболочка сессии (`sessionId`, время создания/обновления записи и номер
   версии) не входит в отпечаток: сравнивается именно полное поле `state` двух
   независимых повторов.

## 3. Отклонённые альтернативы

1. **Сравнение буквального состояния.** Отклонено: реальные отметки времени
   создают ложные различия при одинаковой игре.
2. **Удаление любого свойства `at` или `lastUpdatedAt`.** Отклонено: широкое
   правило способно скрыть значимое игровое расхождение.
3. **Искусственные детерминированные часы вместо реального времени.** Отклонено:
   журнал перестанет достоверно показывать время проведения сессии либо
   потребуется второй параллельный источник времени.
4. **Хеш только публичной клиентской проекции.** Отклонено: совпадение видимой
   части не доказывает одинаковый будущий порядок карт и состояние случайности.
5. **Переиспользование хешей редактора или файлов содержимого.** Отклонено:
   они решают другие задачи и не имеют точных правил канонизации состояния.

## 4. Последствия и ограничения

- Одинаковая последовательность решений получает одинаковый отпечаток, не
  теряя реальное время аудита.
- Нейтральные тесты платформы обязаны проверять точность исключаемых путей,
  порядок ключей и массивов, секретное состояние, отсутствие мутации и формат
  версии.
- Отпечаток подтверждает равенство результата, но не подлинность истории. Для
  защиты от подделки в будущем потребуется подписанный аудиторский конверт.
- Сам отпечаток не доказывает, что использовалась та же версия манифеста. При
  публикационном аудите рядом должны храниться идентификатор игры и хеш
  неизменяемого пакета содержимого.
