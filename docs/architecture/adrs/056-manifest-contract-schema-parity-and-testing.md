## 1. Понимание решения

Решение понято так: JSON Schema остаётся единственным источником истины для структур
манифестов (ADR-025), но это правило сейчас не защищено механически. TypeScript-
контракты в `packages/contracts/*` поддерживаются вручную и уже разошлись со схемой.
Нужно зафиксировать одно направление синхронизации (схема → TS), автоматизировать его
и добавить контрактные тесты, чтобы дрейф становился жёсткой ошибкой сборки, а не
тихой рассинхронизацией.

## 2. Контекст

Ревью проекта (`docs/reviews/2026-06-27-full-project-review.md`) подтвердило, что
ручная парность схем, TypeScript-контрактов и runtime-чтения допускает
расхождения, а неисполняемые заглушки тестов их не обнаруживают. Без
механического направления генерации и drift check парность будет деградировать.

## 3. Термины

- **SSOT (single source of truth)** - единственный источник истины структуры данных.
- **Parity (парность)** - гарантия, что TS-типы точно отражают JSON Schema.
- **Contract test** - тест, проверяющий, что реальные данные (`games/*/*.manifest.json`)
  валидны по схеме, а сгенерированные TS-типы согласованы со схемой.
- **Drift check** - CI-проверка, падающая при расхождении сгенерированного артефакта и
  закоммиченного.

## 4. Принятое решение

1. **JSON Schema остаётся SSOT** структур манифестов (подтверждение ADR-025). Любое
   новое поле сначала появляется в схеме.
2. **Направление синхронизации - схема → TS.** TypeScript-контракты в
   `packages/contracts/*` генерируются из JSON Schema стандартным инструментом
   (например `json-schema-to-typescript`), а не пишутся вручную.
3. **CI drift check** перегенерирует TS из схемы и сравнивает с закоммиченным; любое
   расхождение - ошибка сборки (по образцу authoring compiler drift check).
4. **Контрактные тесты** являются исполняемыми: как минимум AJV round-trip - валидация всех поставляемых
   `games/*/game.manifest.json` и `ui.manifest.json` против схемы, и проверка, что
   типы компилируются над этими данными.
5. Любой выявленный дрейф устраняется в JSON Schema, после чего производные типы
   регенерируются, а runtime-чтение известных полей остаётся типизированным.
6. **Размещение AI-схем** приводится к единообразию: AI JSON Schema эмитируются как
   `.schema.json` в `docs/architecture/schemas/ai/` (генерация из того же TS-источника
   допустима для AI-слоя, где TS уже является носителем схемы), либо это решение явно
   документируется. Цель - чтобы все контрактные схемы имели единое, обнаружимое место.

Обратный TS → schema hook не может конкурировать с выбранным направлением
схема → TS; вспомогательные экспорты допустимы только как производные проверки.

## 5. Граница ответственности

| Артефакт | Роль |
| --- | --- |
| `docs/architecture/schemas/*.schema.json` | SSOT структур |
| `packages/contracts/*/src` (TS) | производное от схемы, не источник |
| CI drift check | гарантия парности |
| Contract tests | гарантия валидности реальных данных по схеме |
| AJV в runtime/compiler | исполнение схемы, не дублирование её в коде |

## 6. Архитектурные инварианты

- JSON Schema - источник истины структур манифестов.
- TS-контракты генерируются из схемы, не пишутся вручную.
- Расхождение TS и схемы - жёсткая ошибка CI.
- Каждое поле, читаемое runtime-ом, типизировано (без `raw`-обхода для известных полей).
- Контрактные пакеты имеют исполняемые тесты, а не `exit 1`-заглушки.
- Валидация структур выполняется стандартным валидатором (AJV), а не ручными
  type guards (ADR-025, anti-pattern «declarative vs imperative»).

## 7. Отклоненные альтернативы

### TS как источник истины, схема генерируется из TS

Отклонено: противоречит ADR-025 (JSON Schema - SSOT) и теряет кросс-языковую,
декларативную природу схемы. Совпадение имён `$defs`/интерфейсов исторически возникло
из такого подхода, но целевое направление - обратное.

### Оставить ручную парность с код-ревью как защитой

Отклонено: `overrides` показывает, что ручная парность уже не выдерживается.

### Не трогать `exit 1`-заглушки, полагаясь на typecheck

Отклонено: typecheck не ловит расхождение TS со схемой и не проверяет валидность
поставляемых данных; именно так дрейф `overrides` остался незамеченным.

## 8. Последствия

- Требуются генератор схема → TS и CI drift check.
- TS-контракты в `packages/contracts/*` становятся генерируемыми; ручные правки
  переносятся в схему.
- Контрактные пакеты имеют исполняемые тесты.
- AI-схемы получают единое место хранения или явно документированное исключение.
- Расхождения схем и производных типов не проходят CI молча.
