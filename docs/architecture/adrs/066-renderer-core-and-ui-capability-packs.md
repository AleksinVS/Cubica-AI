# ADR-066: Декомпозиция рендерера — ядро и жанровые UI capability packs

- Статус: Accepted (2026-07-07)
- Дата: 2026-07-07
- Связанные ADR: ADR-055 (чистота рендерера), ADR-058 (серверные capabilities настольных игр), ADR-062 (Phaser-канал, ленивая загрузка), ADR-064 (headless core + адаптеры каналов, шов player-core)
- Связанные долги: LEGACY-0039 (готовность рендерера к декомпозиции)

## Как понято решение

Единый game-agnostic рендерер `apps/player-web` — правильное решение сегодня,
но с ростом числа типов игр он рискует стать «толстым» узлом: каждый жанр
приносит свои UI-компоненты и каналы, и всё это копится в одном бандле и одной
схеме. Решается вопрос: как заранее зафиксировать ось декомпозиции рендерера —
чтобы при срабатывании триггеров разгрузка была механической, а не
переписыванием — не выполняя саму декомпозицию преждевременно.

## Контекст

- Реестр UI-компонентов сегодня — статический набор из 7 типов, зашитый
  импортами в `apps/player-web/src/components/manifest/ui-component-node.tsx`;
  типы описаны одним закрытым набором в схеме UI-манифеста.
- Сервер уже прошёл эту развилку: ADR-058 ввёл платформенные capabilities
  (`random.roll`, `deck.*`, `turn.*`) как подключаемые наборы возможностей,
  а не монолитное расширение движка. Клиенту нужна симметричная модель.
- Прецедент ленивого тяжёлого канала есть: Phaser (ADR-062) загружается
  лениво и инъецируется в плагин через точку вклада `phaserSceneFactory`.
- Шов ядра уже охраняется CI (ADR-064, `verify:player-core-seam`):
  presenter/lib не зависят от React — «тяжелеет» только слой `components/`.
- Первый кандидат в жанровый пакет виден: «настольный» набор (поле, колода,
  кубики, рука карт) из трека ADR-058.

## Альтернативы

1. **Ничего не фиксировать.** Минус: рост рендерера станет незапланированным
   дрейфом (нарушение правила 9 CLAUDE.md), а декомпозиция под давлением
   сроков — переписыванием.
2. **Декомпозировать сейчас.** Минус: преждевременно — жанровый пакет пока
   один потенциальный, реестр из 7 типов проблемой не является; это ровно тот
   overengineering, от которого проект осознанно уходит (ADR-064).
3. **Плеер на тип игры** (возврат к библиотеке viewers ADR-014). Минус:
   размножает plugin API, схему, конвейер публикации и e2e-обвязку на каждый
   тип; отклонено.
4. **Принято: зафиксировать ось «ядро + жанровые UI capability packs»,
   именованные триггеры и объективный датчик (бюджет бандла) — декомпозицию
   выполнять по триггеру.**

## Решение

1. **Модель декомпозиции.** Рендерер делится по оси «ядро + жанровые UI
   capability packs» (пакет возможностей — именованный набор платформенных
   UI-компонентов одного жанра игр):
   - **ядро**: layout, экраны и роутинг, декларативная привязка действий,
     резолвер выражений, панели, базовые компоненты (текст, кнопка,
     изображение, переменная);
   - **жанровые пакеты**: например «настольный» (поле, колода, кубики, рука
     карт) и «симуляционный» (Phaser-поверхность `simulationSurface` —
     фактически первый существующий пакет).
2. **Декларация в манифесте.** UI-манифест объявляет требуемые пакеты;
   использование компонентов необъявленного пакета — ошибка валидации схемы.
   Загрузка пакетов ленивая, по объявлению.
3. **Механизм — реестр вместо switch.** Статическая диспетчеризация типов
   компонентов заменяется реестром «тип → рендерер компонента». Реестр
   вводится вместе с первым реальным жанровым пакетом, не раньше.
4. **Каналы — ленивые острова.** Правило из прецедента Phaser (ADR-062)
   становится нормой: тяжёлая канальная поверхность загружается лениво и
   подключается через точку вклада; DOM-канал не платит за остальные.
5. **Схема декомпозируется той же осью.** Жанровый пакет приносит свой
   композируемый модуль схемы UI-компонентов; базовая схема описывает только
   ядро и механизм подключения. Схема остаётся SSOT (ADR-025/056).
6. **Конформанс-фикстура на пакет.** Каждый жанровый пакет приходит с
   минимальной фикстурной игрой в canonical verification — по образцу
   `games/simple-choice/` (ядро) и `games/ai-driven-choice/` (AI-контур).
7. **Объективный датчик вместо интуиции.** Вводится CI-гейт бюджета бандла
   `player-web` (first-load JS): текущий размер фиксируется как базовая
   линия, превышение порога — ошибка CI и сигнал триггера декомпозиции.
   Исполнительная задача — `TSK-20260707-player-web-bundle-budget`.
8. **Триггеры декомпозиции** (любой из): второй жанровый набор компонентов;
   срабатывание бюджета бандла; третий канал доставки.
9. **Границы чистоты сохраняются.** Пакеты — платформенные и жанрово-общие
   (правило 10 CLAUDE.md): никаких game-specific компонентов в пакетах;
   специфика конкретной игры остаётся в её манифестах и плагине
   (ADR-037/039/055).

## Последствия

- Положительные: рост рендерера становится управляемым и измеримым; при
  срабатывании триггера декомпозиция — механический перенос компонентов в
  пакет за готовым реестром; клиентская модель симметрична серверной
  (ADR-058), что упрощает проектирование будущих жанров.
- Риски: преждевременное дробление на пакеты ради «красоты» — запрещено
  (декомпозиция только по триггерам); бюджет бандла может ложно срабатывать
  на инфраструктурные обновления (Next.js) — порог пересматривается явным
  изменением базовой линии в коммите, а не отключением проверки.
- Долги: LEGACY-0039 переходит из «зафиксировать направление» в «исполнить
  по триггеру согласно ADR-066».
