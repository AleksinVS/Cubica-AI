## 1. Понимание решения

Решение понято так: общий доступный интерфейс действий на интерактивном поле
должен уметь принять обычную строку так же, как он уже принимает выбор из
списка и число. Это общий способ ввода параметра, а не отдельная форма
конкретной игры.

## 2. Контекст

Серверный контракт Game Intent уже поддерживает ограниченные строковые
параметры и проверяет их по JSON Schema. Общая DOM-проекция действий
(обычные элементы web-формы, доступные с клавиатуры и вспомогательных
технологий) умеет показать только список и число. Игровому плагину поэтому
пришлось бы либо ограничить разрешённый свободный текст, либо создать
параллельную игровую форму вне общего пути действия.

## 3. Решение

Закрытый тип `AccessibleBoardActionField` получает вариант `kind: "text"`.
Проекция поля содержит:

- стабильное имя параметра;
- видимую подпись;
- признак обязательности;
- необязательное начальное строковое значение;
- необязательные подсказки `minLength`, `maxLength` и `pattern`.

Общий host отображает этот вариант стандартным управляемым
`input type="text"`: текущее значение всегда берётся из временного черновика
действия, а каждое изменение сразу обновляет этот черновик. Пустое
необязательное значение не отправляется. Строка передаётся без скрытого
обрезания пробелов, изменения регистра или другой предметной нормализации.

Добавление поля не превращает host в автоматический построитель всех форм из
JSON Schema. Игровой плагин по-прежнему выбирает, какие уже разрешённые
действия и поля уместны в текущем экранном состоянии, а общий host отвечает
только за нейтральный ввод и отправку.

## 4. Граница доверия и доступность

- `paramsSchema` опубликованного Game Intent остаётся единственным доверенным
  валидатором типа, длины, шаблона и обязательности.
- Ограничения в UI являются ранними подсказками браузеру и не расширяют
  серверный контракт. Клиент не может сделать недопустимую строку допустимой.
- Поле имеет программно связанную подпись, работает с клавиатуры и использует
  стандартную семантику текстового ввода.
- Начальное и текущее значения всегда являются строками; компонент не
  переключается между управляемым и неуправляемым режимами.
- Временный черновик не сохраняется как состояние партии и не принимает
  игровых решений.
- Общий host и его тесты не содержат идентификаторов, подписей или правил
  конкретной игры.

## 5. Отклонённые альтернативы

1. **Использовать только заранее подготовленные варианты.** Отклонено:
   свободный строковый параметр теряет предусмотренный продуктом смысл.
2. **Создать игровую React-форму.** Отклонено: появляется второй путь ввода,
   отправки и доступности для одного и того же Game Intent.
3. **Сразу автоматически строить все формы из JSON Schema.** Отклонено для
   этого изменения как существенно более широкая задача; поле текста
   дополняет уже действующий ограниченный контракт.
4. **Доверять клиентской проверке длины или шаблона.** Отклонено: клиент не
   является границей безопасности.

## 6. Последствия

- Игры могут использовать свободные строковые параметры на map-first экране
  без игрового исключения в общем приложении.
- Поле остаётся доступным при отказе визуальной canvas-поверхности, поскольку
  живёт в обычной DOM-проекции действий.
- API плагина, host-компонент и нейтральные компонентные тесты должны
  поддерживать новый закрытый вариант согласованно.
