# План миграции: Bounded Manifest-Driven Gameplay Mechanics для Антарктики

**Статус**: Актуален
**Компоненты**: `games/*`, `services/runtime-api`, `packages/contracts/manifest`, `apps/player-web`

## Контекст

При миграции игры "Антарктика" в канонический manifest-driven runtime возникла необходимость добавления реальных gameplay-механик. Вместо создания универсального workflow-движка (который избыточен на текущем этапе для Антарктики), было принято решение моделировать механики явными (explicit) переходами в JSON-манифесте. Этот документ описывает правила такой миграции.

## Правила миграции механик

1. **Явные действия (Explicit Actions)**
   - Игровые действия описываются вручную в манифесте с явными id и детерминированными метаданными (`deterministic`).
   - Переходы, действия подтверждения (confirm/advance) и пути следования (follow-up paths) должны быть явными, а не скрытыми сайд-эффектами рантайма.

2. **Детерминированный рантайм вместо LLM**
   - Для "Антарктики" целевой LLM-движок избыточен. Детерминированный рантайм на основе JSON-манифеста является легальным плановым решением.
   - Специфические механики (прогрессия на основе пороговых значений, переключение веток, разблокировка карточек) допустимы как локальная семантика манифеста для конкретных игр.
   - Универсальный workflow engine, selector DSL, интерпретатор правил или роутер веток не вводятся до появления повторяющихся кейсов между разными играми.

3. **Аудируемость (Auditability) прогресса**
   - Все, что требуется UI-плееру для отображения доступности, прогресса или выбранного состояния, должно проецироваться в явное детерминированное состояние (обычно в `state.public`).

4. **Канонические маршруты**
   - Цели ветвления и идентификаторы веток в рантайме должны опираться на канонические строковые идентификаторы (canonical string ids), а не на числовые индексы из легаси (legacy numeric line indexes).

## Документирование этапов (Slices)

Детали конкретных этапов (step-, board-, card- specific rules) миграции "Антарктики" должны фиксироваться в **Gameplay Slice Records** в каталоге `docs/architecture/gameplay-slices/`.
