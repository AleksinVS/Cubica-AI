# Черновик: WebSocket протокол Router (Фаза 2+)

## Назначение
Документ фиксирует целевое направление для real-time канала (WebSocket — протокол для постоянного двустороннего соединения) между SDK и Router.

Для MVP используется HTTP `POST /submit` (см. `docs/architecture/router-openapi.yaml`).

## Минимальные события
- `session.join` — клиент присоединился к сессии.
- `session.update` — сервер отправил обновление состояния (полное состояние или патч).
- `session.error` — управляемая ошибка обработки действия.

## Формат полезной нагрузки
- Базовый патч: JSON Merge Patch (RFC 7396).
- Опционально: JSON Patch (RFC 6902).
