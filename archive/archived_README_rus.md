# Игровая платформа Cubica

Монохранилище генеративной игровой платформы Cubica. Проект объединяет сервисы, SDK, шаблоны и документацию, необходимые для поставки MVP, описанного в GAME_PLATFORM_IMPLEMENTATION_PLAN.md.

## Начало работы
- Ознакомьтесь с GAME_PLATFORM_IMPLEMENTATION_PLAN.md, чтобы понять план поставки.
- Изучите PROJECT_STRUCTURE.md для краткого обзора каталогов.
- Просмотрите game_platform_architecture.md, чтобы разобраться в целевой архитектуре.



### Разработка в Windows
- Держите рабочую копию в `C:\Work\Tallent\Cubica` и запускайте инструменты через PowerShell (`pwsh`).
- Используйте `pwsh ./scripts/dev/bootstrap.ps1 -InstallDependencies -SeedData`, чтобы подготовить зависимости и тестовые данные.
- Перед запуском сервисов убедитесь в готовности инструментов командами `node -v`, `python -m pip --version` и `docker compose version`.
- Перед переключением веток фиксируйте изменения или откладывайте их, чтобы рабочая копия оставалась чистой.

### WSL (архив)
- Настройка WSL сейчас приостановлена; при возобновлении поддержки обратитесь к `docs/legacy/dev-environment-wsl.md`.

## Рабочий процесс разработки
- После первого пуша `main` становится защищённой веткой: запретите прямые коммиты, потребуйте статус-проверки и минимум одно одобрение; настройте правило в параметрах репозитория > Branches > Add rule для `main`.
- Работу над функциональностью ведите в ветках `feature/<component>-<topic>` (для срочных исправлений используйте `hotfix/<component>-<topic>`); держите ветки краткоживущими.
- Перед PR выполните ребейз на актуальный `main`, локально запустите тесты и линтеры, а затем `pwsh -File scripts/ci/validate-legacy.ps1` (или кроссплатформенную версию `python scripts/ci/validate-legacy.py`).
- GitHub Actions в `.github/workflows/ci.yml` запускает оба валидатора при каждом push/PR.
- Локальные скрипты подготовки: `scripts/dev/bootstrap.ps1` (PowerShell) и `scripts/dev/bootstrap.sh` (bash) устанавливают зависимости и загружают фикстуры.
- Требования, обязательные проверки и роли описаны в `docs/processes/review-policy.md`.
