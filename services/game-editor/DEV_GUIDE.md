# Руководство по разработке: Game Editor

## Назначение
- Предоставляет авторам рабочее пространство для создания и итерации игр с LLM-помощником.
- Обеспечивает визуализацию, чат-редактирование и предпросмотр без угроз целостности данных.

## Объем MVP
- React-приложение с чат-интерфейсом, отправляющим intents в Engine/LLM.
- Только чтение диаграмм потоков из JSON Repository; drag-and-drop появится позже.
- Экран предпросмотра использует тот же рендеринг, что и клиентские приложения.

## API
- Работает с REST-эндпоинтами Repository для CRUD игровых манифестов (`docs/architecture/repository-openapi.yaml`).
- Интегрируется с authoring-эндпоинтами Engine для быстрой симуляции сценариев.
- Планируется подключение аутентификации Portala и телеметрии Editor (`docs/architecture/editor-telemetry.md`).

## Config
- `.env` параметры: URL Repository/Engine, ключи авторизации, фиче-флаги режимов редактирования.
- Feature-флаги управляют провайдером LLM, режимами предпросмотра, наличием экспериментов; актуальные значения фиксируются здесь.
- Скрипт `scripts/dev/editor-sync.ps1` используется для локальной синхронизации контента и должен выполняться перед демо.

## Testing
- Unit-тесты компонентов (React Testing Library) с фикстурами Repository.
- Интеграционные сценарии «редактирование → сохранение → валидация» через mock LLM.
- Визуальные регрессионные тесты диаграмм и экранов предпросмотра.

## Legacy
- Временные ограничения UX (ручное обновление, отсутствие undo и т.п.) фиксируются в этом документе и `docs/legacy/editor-debt.md`.
- Все заглушки и мок-адаптеры регистрируются в `docs/legacy/debt-log.csv` с целевой фазой снятия.
- До готовности auth используются сервисные ключи; запись в реестре долга обязательна.

## Документация и обновления
- Обновлять гид по итогам каждой итерации (новые API, UX-изменения, зависимости).
- Поддерживать библиотеку промптов в `docs/processes/editor-prompts.md` и будущий онбординг дизайнеров.
- Ссылки на постмортемы и аналитические отчёты добавлять в раздел Legacy при появлении.
