# Руководство по разработке: Metadata Database

## Назначение
- Консолидирует аналитические и операционные метаданные об играх, сессиях и процессах авторинга.
- Предоставляет инсайты для дизайнеров, эксплуатации и бизнеса.

## Объем MVP
- Проектирование схемы стартует во Фазе 2.
- Первая поставка во Фазе 3: кластер PostgreSQL/Timescale с базовыми BI-коннекторами.
- Ранние фазы используют экспорт из Repository и Router, размещённый в `docs/legacy/`.

## API
- Экспортирует SQL, материализованные представления и API для аналитических сервисов (дизайн описывается по мере готовности).
- Загружает данные из логов Router, транскриптов Engine, истории Repository и событий Portala.
- Все интеграции синхронизируются с дорожной картой `docs/processes/analytics-roadmap.md`.

## Config
- Планируемые параметры: настройки ingestion jobs, политики retention, секционирование, настройки Timescale.
- До Фазы 3 конфигурации каталога документируются в этом файле и сопровождаются ADR.
- Требования к безопасности (GDPR, анонимизация) фиксируются заранее и привязываются к соответствующим веткам.

## Testing
- Тесты миграций схем (sqitch/flyway) и проверка целостности данных после каждой поставки.
- Unit/Integration тесты ingestion jobs, мониторинг аномалий на агрегатах.
- Бенчмарки производительности запросов при изменениях индексов/структур.

## Legacy
- Временные скрипты и ручные отчёты фиксируются в `docs/legacy/metadata-stubs.md` и реестре долга.
- Все заглушки в ingestion/экспорте регистрируются в `docs/legacy/debt-log.csv` с целевой фазой.
- Переход от файловых экспортов к стриминговой загрузке оформляется отдельным ADR.

## Документация и обновления
- Поддерживать ER-диаграммы и словари таблиц в `docs/architecture/metadata/`.
- Добавлять онбординг аналитиков и data-инженеров по мере готовности.
- Инциденты с данными фиксируются в `docs/legacy/metadata-incidents.md`.
