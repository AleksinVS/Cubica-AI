# Antarctica (карточная симуляция)

Этот каталог содержит автономный HTML-прототип обучающей игры про экспедицию в Антарктику. Игрок управляет командой и принимает решения по карточкам, которые влияют на показатели экспедиции и итоговый счёт.

## Как запустить
Откройте `Game.html` напрямую в браузере (двойной клик по файлу или `file:///.../draft/Antarctica/Game.html`). Для работы не требуется сервер: все зависимости лежат локально в каталоге `draft/Antarctica`.

## Геймплей
- На экране выводится панель показателей и текущее название этапа.
- Игровой прогресс идёт по заранее заданному сценарию (таймлайн — упорядоченный список карточек и экранов-информаций). Игрок нажимает на карточки, читает их оборот и выбирает «Далее» там, где это разрешено.
- В некоторых шагах предлагается выбрать команду: отображаются карточки персонажей, из которых нужно собрать группу. Эффекты выбранных персонажей сразу применяются к показателям.
- Часть шагов открывается как модальные окна (`type: info`) — они блокируют дальнейшие действия, пока игрок не нажмёт «Далее».
- Итоговый счёт рассчитывается как `60 - time`, где `time` накапливается из действий карточек и команды.

## Показатели
Все показатели отображаются в верхней навигации и в модальном окне истории. Названия в коде:
- `pro` — прогресс экспедиции.
- `rep` — репутация.
- `lid` — лидерство команды.
- `man` — управление/организация.
- `stat` — статус/стратегия (в оригинале — «стат»).
- `cont` — контракты или контакты.
- `constr` — строительство/инфраструктура.
- `time` — суммарные затраты времени; от него считается счёт (`score`).

## Структура данных сценария
Главные данные определены в `Game.html`:
- `cardsObj` — объект со всеми карточками и экранами. Ключ — идентификатор (число или строка), значение — описание.
  - `type`: `cards` (обычная карточка с лицевой и оборотной стороной), `team` (карточка члена команды), `info` (модальное окно без выбора).
  - `f` / `b`: содержимое лицевой (`front`) и оборотной (`back`) сторон для `cards`, включая `text`, `action` (список эффектов), признак `locked`, флаг `go` для отображения кнопки «Далее».
  - Для `info`: `title`, `text`, `bg` (картинка), `locked` (блокирует карточку, т.е. ее нельзя выбрать до наступления определенных условий).
  - Для `team`: `name`, `img`, `text`, `action` (действия при добавлении персонажа).
- `timeline` — вложенный массив, задающий последовательность шагов. В нём перечислены блоки, каждый содержит список идентификаторов карточек, которые выводятся одновременно.
- `log` — список уже открытых карточек.
- Ряд вспомогательных функций (`f25_30`, `unlock39`, и т.п.) выполняют команды при определенных условиях, например, автоматически разблокируют карточки при выполнении условий.

timeline - трехмерный массив: 
 первое измерение - сюжетные линии 
 второе - этапы игры (шаги) 
 третье - элементы cardsObj: наборы карточек, страницы, переходы или условия на весь этап игры
 переход - содержит условие перехода на другую сюжетную линию
 переходs и условия на этапы должны стоять первыми в массиве
 пример перехода
` 'g1':{'type':'jump','alt':[{'pro':[{'>':[40,'func',goto,{1,15}]}]}]}}};`
примеры условий:

`'rep':[{'<':[15,'rep',-3]}],` - если репутация < 15 то репутация -3
`'log':[{'out':['leo','time',1]}]` - если Леонид в списке действий и членов команды то время +1 (ходы и команда пишутся в один массив) 

параметр "go" -  определяет будет ли у карточки кнопка "Далее" (для лицевой стороны не реализован)

### Формат эффектов (`action`)
Эффекты карточек задаются массивом записей. Каждая запись обрабатывается `game.operate()`:
- `[<metric>, <число>]` — добавить число к показателю.
- `[<metric>, '>', <порог>, <действие>]` / `[<metric>, '<', ...]` / `[<metric>, '==', ...]` — условие; при выполнении рекурсивно выполняется вложенное действие.
- `[<metric>, '=', <максимум>]` — ограничить показатель сверху.
- `['log', 'in'|'out', <cardId>, <действие>]` — проверка того, попадала ли карточка в историю.
- `['func', <ссылка-на-функцию>, <аргумент>]` — вызвать произвольную функцию (например, `goTo` или `unlock`).

 


## Поток управления
Основная логика находится в конце `Game.html`:
- `game.next()` двигает курсор по `timeline`, собирает карточки текущего шага и рендерит нужный шаблон: сетка карточек (`type: cards`), выбор команды (`type: team`) или инфо-модалка (`type: info`).
- `game.operateArray()` и `game.operate()` применяют эффекты из `action`, включая условия и вызовы вспомогательных функций.
- При клике на карточку её эффекты применяются, результат фиксируется в `log`, а шаг записывается в историю (`#modal-history`), где видно, как изменились показатели.
- `setVars()` синхронизирует навигационную панель со значениями внутри объекта `game`.
- Навигационный пункт «История» открывает модалку с хронологией всех разыгранных карточек и их влиянием.

## Ассеты и зависимости
- UI построен на Bootstrap 3 (`dist/css/bootstrap.min.css`, `dist/js/bootstrap.min.js`) и jQuery (`jquery.min.js`). В комплекте есть `ie10-viewport-bug-workaround.js`.
- Плагин `jquery.flip.min.js` подключён для эффекта переворота карточек (используется в CSS-анимациях).
- Пользовательские стили — `penguines.css` и `justified-nav.css`. Изображения для карточек лежат в `img/`, иконка — `favicon.ico`.
- Все файлы поставляются вместе с HTML; внешних сетевых зависимостей нет.

## Как расширять сценарий
1) Добавить карточку:
   - Создайте запись в `cardsObj` с нужным `id`.
   - Для `type: cards` задайте `f` (лицевая часть) и `b` (оборот) и массив `action`.
   - Для `type: info` добавьте `title`, `text`, `bg`; при необходимости `locked: true`, чтобы заблокировать окно.
2) Включить карточку в прохождение:
   - Добавьте `id` в соответствующий блок `timeline` в том порядке, в котором ее нужно показать.
   - Если карточка должна появляться по условию (например только после других), используйте `locked: true` и разблокирующую функцию (`unlock`) или условные `action`.
3) Настроить эффекты:
   - Используйте формат `action`, чтобы изменять показатели, проверять условия (`>`, `<`, `==`) или вызывать вспомогательные функции (`func`).
   - При работе с временем помните, что счёт игры — `60 - time`.
4) Обновить ресурсы:
   - Для новых картинок добавьте файл в `img/` и укажите имя в карточке.

## Известные особенности
- Код хранит часть текста в cp1251/UTF-8 без явной конвертации, поэтому в некоторых инструментах могут отображаться «кракозябры». В браузере с правильной кодировкой текст читается корректно.
- Переменные для выбора команды (`teamcount`, `skills`) используются как глобальные и не инициализируются явно; движок браузера создаёт их динамически. При доработке стоит объявлять такие переменные явно, чтобы избежать скрытых ошибок.
- Логика рассчитана на работу в одном файле без сборки; любые новые зависимости нужно добавлять локально рядом с `Game.html`.
