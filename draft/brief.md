1. Манифест создается на основании описания игры (файлы правил/сценариев/методических материалов). Файлы описания - источник правды на естественном языке, Манифест - технический документ переводящий правила на удобный для алгоритмической обработки язык. При заполнении манифеста создаются ссылки на описание игры (файлы правил/сценариев/методик) и соответствующие "якоря" в описаниях. Манифест изменяется через изменение описания игры, т.е. при изменении описания обновляется манифест. Если в описании указано, что у карточки должен быть определенный текст, то этот текст вносится в манифест вместе со ссылкой на блок текста в файле сценария. Если для текста нужно расширенное оформление, то текст храниться в формате одного из языков разметки, например, HTML.

Правило «по умолчанию»: всё смысловое текстовое содержимое — только в описаниях; манифест — структура, ссылки и короткие машинные строки (id, ключи, enum‑значения). Если понадобятся «копии» текста в манифесте, делаем это как автоматическую, всегда перегенерируемую проекцию из описаний.

Как это будет отображаться и рассчитываться храним в скриптах и файлах с описанием UI. Т.е. например, в манифесте мы указываем что есть последовательность экранов, есть сущность "экран", у нее есть определенные параметры и составляющие, все это описывается в манифесте в абстрактных игровых терминах. В манифесте в узле экрана есть ссылка на файл с js-скриптами для обработи событий экрана и ссылка на класс в файле с описанием UI по которому отрисовываются все экраны такого типа. Пакеты расширения для Viewers и Game engine


1. Описываем архитектуру Game Editor.

2. Создаем MVP Game Editor (боковая панель чата и поле редактора):

      a. Подготовительный этап - диалог в чате с ИИ-агентом.
            - Определить бизнес-цели игры (на что нужно повлиять в работе);
            - Определить учебные цели игры (какие выводы должен сделать игрок, какие инструменты отработать и проч.);
            - Подобрать игровые действия (механики), чтобы можно было выполнить учебные цели;
            - Выбрать жанр игры и написать сюжет игры.
      Для этого этапа нужно подготовить skills и промты для ИИ-агента, чтобы он мог эффективно направлять пользователя на этом этапе.

      b. Сценарий и правила игры. При работе над сценарием и правилами игры пользователь общается с ИИ-агентом в чате, а поле редактора - простой текстовый редактор создаваемых файлов. Агент создает файлы, пользователь имеет возможность их отредактировать. Сценарий также может отображаться в виде flow-диаграммы в окне редактора - для этого нужно продумать где и как хранить данные (структуру и свойства) flow-диаграммы в манифесте, чтобы не создвать их каждый раз динамически.

      c. Интерфейс игры. При создании интерфейса экрана игры в поле радактора отображается мокап экрана, на катором можно выделить отдельные элемены (мокап предварительно будет размечен в json). Пользователь может выделить элемент (щелчком мыши) или область мокапа (растягиванием прямоугольника) и попросить LLM  изменить выделенный элемент или область. При выделении элемента/области рядом с курсором мыши появляется активное поле ввода текста в которое можно сразу начать печатать запрос к LLM.
      
      d. Прототип игры. После того, как мокап создан и утвержден, LLM-агент делает верстку экрана игры и отображает сверстанную (собранную) страницу в поле редактора. Пользователь может выбрать элементы и указать агенту, как их нужно изменить.

      e. Отладка. Для отладки, игра запускается в окне редактора, при этом окно чата становится сворачиваемым.
