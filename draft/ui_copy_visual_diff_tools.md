# Инструменты для точного копирования UI агентами (Visual Diff Loop)

При задаче точного переноса UI из эталона (например, `draft/game-player-nextjs`) в целевой проект (например, `games/antarctica`) со строгим сохранением существующей архитектуры и DOM, AI-агенты часто терпят неудачу, если полагаются только на "зрение" (Vision-модели). 

Для решения этой проблемы применяется подход **Visual Diff Loop** (Цикл визуального сравнения), который ограничивает агента и дает ему точную математическую или попиксельную обратную связь.

## Экосистема Gemini Testing / Testplane

Проект `gemini-testing` (ныне развивающийся как **Testplane**, ранее Hermione) предоставляет отличный набор инструментов для построения такого цикла:

1. **`testplane-mcp` и `testplane-skill`**: 
   - Инструменты для интеграции AI-агентов (LLM) с браузерами через фреймворк Testplane (Model Context Protocol).
   - Позволяют агенту программно инспектировать DOM и извлекать точные стили (Computed Styles), а не угадывать их по картинке.
2. **`looks-same`**: 
   - Низкоуровневая Node.js библиотека для попиксельного сравнения изображений (Visual Diffing).
   - Умеет игнорировать сглаживание шрифтов (anti-aliasing) и незначительные артефакты рендеринга. Используется для сравнения скриншота, сделанного агентом, с эталоном.
3. **`testplane` (Browser Test Runner) и `html-reporter`**: 
   - Мощный фреймворк на базе WebdriverIO для автоматизации браузера и визуального регрессионного тестирования.
   - Позволяет делать скриншоты отдельных компонентов и выдает наглядные HTML-отчеты с подсвеченными красным различиями.

## Как работает Visual Diff Loop для AI-агента

Чтобы заставить агента точно перенести стили без изменения архитектуры:

1. **DOM-Lock (Ограничение в промпте)**: Агенту строго запрещается менять иерархию HTML/React компонентов в целевом проекте. Ему разрешено менять только CSS, классы Tailwind или styled-components.
2. **Baseline (Эталон)**: Testplane делает скриншоты ключевых компонентов эталонного проекта.
3. **Итерация агента**: Агент анализирует эталон (идеально — через `browser_evaluate` достает точные цвета, отступы, шрифты `getComputedStyle()`) и применяет их к целевому проекту.
4. **Проверка (Visual Diff)**: Testplane делает скриншот целевого проекта после правки агента. Скрипт (`looks-same`) сравнивает его с эталоном.
5. **Feedback (Обратная связь)**: Если есть разница, агент получает результат сравнения (heatmap или текстовое описание расхождений) и делает следующую итерацию, пока разница не станет минимальной.
