## Назначение

Этот файл хранит только повторяющиеся ошибки, которые меняют поведение навыка.
Он читается при спорной ситуации или обновлении навыка, но не является
обязательным контекстом каждой UI-задачи.

## Действующие уроки

1. Один тяжёлый процесс для локальной правки и целого экрана заставляет агентов
   обходить правила. Масштаб нужно выбирать явно.
2. Статичное изображение не определяет действия, состояния и мобильное
   поведение. Наблюдение, вывод и решение PM должны различаться.
3. Контраст пикселей помогает найти деталь, но не понимает её назначения.
4. Полный SHA скрипта создаёт ложный дрейф при изменении комментариев; профилю
   нужны версии контрактов и алгоритмов.
5. Несколько общих UI-навыков с широкими триггерами дают конфликт маршрутизации;
   `ui-from-mockup` должен оставаться владельцем реализации по образцу.

## Правило обновления

Добавлять урок после повторившейся ошибки или подтверждённой практики. Указать
наблюдаемое последствие и изменение процесса. Не записывать разовую деталь
конкретного макета.
